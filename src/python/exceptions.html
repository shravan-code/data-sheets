Python exceptions

Exceptions

Error handling patterns and custom exceptions.

try/except/finally

try:
    result = x / y
except ZeroDivisionError:
    result = 0
except (TypeError, ValueError) as e:
    print(f"Error: {e}")
    raise
else:
    print("No exception occurred")
finally:
    print("Always executes")

Common Exceptions

ExceptionWhen it occurs
ValueErrorWrong value (e.g., int("abc"))
TypeErrorWrong type operation

IndexErrorList index out of range
AttributeErrorAttribute not found
FileNotFoundErrorFile doesn't exist

Raising Exceptions

# Raise with message
raise ValueError("Invalid input value")

# Re-raise current exception
try:
    dangerous_operation()
except:
    log_error()
    raise  # Re-raises same exception

# Raise from (chaining)
raise RuntimeError("Failed") from original_error

Custom Exceptions

class ValidationError(Exception):
    def __init__(self, message, field=None):
        self.field = field
        super().__init__(message)

# Usage
raise ValidationError("Name required", field="name")

Best Practices

Do:
  • Be specific - catch exact exceptions
  • Let exceptions propagate when caller should handle
  • Use contextlib.suppress() for silent ignoring
Don't:
  • Never catch bare except:
  • Avoid catching Exception unless re-raising
# Silent ignore
from contextlib import suppress
with suppress(FileNotFoundError):
    os.remove("file.txt")
