Performance

This section covers Pandas performance optimization techniques including vectorization, efficient operations, and best practices for fast data processing.

Vectorization - The Key to Speed

Vectorization means applying operations to entire arrays at once, rather than using loops. Pandas uses NumPy under the hood, which provides highly optimized C implementations for vectorized operations.

Why Vectorization is Fast

  • No Python loops: Operations happen in compiled C code
  • Single operation overhead: One call to C vs. millions of Python calls
  • SIMD instructions: Modern CPUs process multiple values simultaneously
  • Memory locality: Data is processed in contiguous blocks

Vectorized vs. Loop

import pandas as pd
import numpy as np

# Create sample data
df = pd.DataFrame({'salary': np.random.randint(50000, 150000, 100000)})

# SLOW: Loop-based approach
def add_bonus_loop(salary):
    result = []
    for s in salary:
        result.append(s * 0.1)
    return result

df['bonus_loop'] = add_bonus_loop(df['salary'])

# FAST: Vectorized approach
df['bonus_vectorized'] = df['salary'] * 0.1
Tip: Vectorized operations are typically 100-1000x faster than Python loops. Always prefer vectorized operations when possible.

Avoiding iterrows()

iterrows() is one of the slowest ways to iterate over a DataFrame. Avoid it whenever possible.

The iterrows Problem

# VERY SLOW: iterrows
for idx, row in df.iterrows():
    df.loc[idx, 'bonus'] = row['salary'] * 0.1

# SLOW: apply with axis=1
df['bonus'] = df.apply(lambda row: row['salary'] * 0.1, axis=1)

# FAST: Vectorized assignment
df['bonus'] = df['salary'] * 0.1

Alternatives to iterrows

# For conditional logic: use np.where or .loc
df['category'] = np.where(df['salary'] > 100000, 'High', 'Low')
df.loc[df['salary'] > 100000, 'category'] = 'High'
df.loc[df['salary'] <= 100000, 'category'] = 'Low'

# For complex conditions: use .query
df['category'] = 'Medium'
df.loc[df['salary'] > 100000, 'category'] = 'High'
df.loc[df['salary'] < 70000, 'category'] = 'Low'

# For aggregation: use groupby
df.groupby('dept')['salary'].mean()

# For custom functions: use .apply (without axis=1)
df['bonus'] = df['salary'].apply(lambda x: x * 0.1)

When iterrows Might Be Acceptable

# Only for very small DataFrames where readability matters most
# and performance is not a concern

for idx, row in small_df.iterrows():
    print(f"{row['name']}: {row['salary']}")

Efficient Selection Methods

Choosing the right selection method can significantly impact performance.

Selection Method Comparison

MethodSpeedUse Case
df['col']FastestSingle column selection
df[['col1', 'col2']]FastMultiple columns
df.locFastLabel-based selection
df.ilocFastPosition-based selection
df.atFastest*Single cell by label
df.iatFastest*Single cell by position
df.queryFastComplex boolean expressions
df[bool_vec]FastBoolean indexing

*For single cell access, at/iat are ~100x faster than loc/iloc

Use at/iat for Single Values

# For single cell access, use at/iat
value = df.at['row_label', 'col_label']      # Fastest for single value
value = df.iat[0, 1]                         # Fastest for position

# Avoid loc for single values when at is available
value = df.loc['row_label', 'col_label']      # Slower

Efficient Boolean Selection

# Create boolean mask once and reuse
mask = df['salary'] > 80000
df_filtered = df[mask]

# Multiple conditions - create mask and combine
mask1 = df['salary'] > 80000
mask2 = df['dept'] == 'Engineering'
mask3 = df['active'] == True

# Combine masks (faster than multiple indexing)
result = df[mask1 & mask2 & mask3]

Use Appropriate dtypes

Correct dtypes not only save memory but can also improve performance.

Performance Impact of dtypes

# Use category for groupby and value_counts (much faster)
df['dept'] = df['dept'].astype('category')
df.groupby('dept')['salary'].mean()        # Faster with category

# Use nullable Int for integer columns with NaN
df['age'] = df['age'].astype('Int8')        # Better than float with NaN

# Use bool instead of int for boolean columns
df['active'] = df['active'].astype(bool)   # Faster boolean operations

Numeric Operations

# Use float32 instead of float64 when precision allows
df['salary'] = df['salary'].astype('float32')

# float32 operations can be faster on some systems
# especially with large datasets

# Use int8/int16 for small integer values
df['age'] = df['age'].astype('int8')

Use eval() and query()

eval() and query() use string expressions and can be faster for complex operations.

eval() for Column Operations

# Traditional approach
df['total'] = df['salary'] + df['bonus']

# With eval (can be faster for many columns)
df.eval('salary + bonus', inplace=True)

# Assign using eval
df.eval('total = salary + bonus', inplace=True)

# Multiple operations
df.eval('gross = salary + bonus; tax = gross * 0.3', inplace=True)

query() for Filtering

# Traditional approach
df[(df['salary'] > 80000) & (df['dept'] == 'Engineering')]

# With query (more readable and often faster)
df.query('salary > 80000 and dept == "Engineering"')

# Using local variables
threshold = 80000
df.query('salary > @threshold')

# Using multiple conditions
df.query('salary > 80000 or (dept == "HR" and salary > 70000)')

When to Use eval/query

# Best for:
# - Many columns in one expression
# - Complex boolean expressions
# - Memory-efficient operations (temp arrays not created)

# Not needed for:
# - Simple single operations
# - Operations on single columns
# - When readability is more important
Performance Note: eval() and query() use numexpr library internally, which can significantly speed up operations on large DataFrames with many columns.

Efficient GroupBy Operations

Use Appropriate Aggregation

# Named aggregation (recommended)
df.groupby('dept').agg(
    avg_salary=('salary', 'mean'),
    max_salary=('salary', 'max')
)

# String aggregation (faster)
df.groupby('dept')['salary'].agg(['mean', 'max', 'min'])

# Single function (fastest)
df.groupby('dept')['salary'].mean()

Use transform for Group Operations

# Add group mean as new column (transform)
df['dept_avg'] = df.groupby('dept')['salary'].transform('mean')

# NOT this way (creates many small DataFrames)
dept_means = df.groupby('dept')['salary'].mean()
df['dept_avg'] = df['dept'].map(dept_means)

Categorical groupby

# Convert to category for faster groupby
df['dept'] = df['dept'].astype('category')
df.groupby('dept')['salary'].mean()  # Faster with category

Efficient Merge Operations

Prepare DataFrames Before Merge

# Sort by join key (can speed up merge)
df1 = df1.sort_values('key')
df2 = df2.sort_values('key')

# Set index when joining on index
df1 = df1.set_index('key')
df2 = df2.set_index('key')
result = df1.join(df2)  # Index-based join is fast

Broadcast Small Tables

# For small lookup tables, convert to dict for fast lookup
lookup = df_small.set_index('id')['name'].to_dict()
df['name'] = df['id'].map(lookup)

# This is often faster than merge for simple lookups

Merge vs Map

# For single column lookup, map is faster
lookup = other_df.set_index('key')['value'].to_dict()
df['value'] = df['key'].map(lookup)

# Use merge when you need multiple columns from other_df
result = df.merge(other_df, on='key', how='left')

String Operations Performance

Vectorized String Operations

# Always use .str accessor (vectorized)
df['name'].str.upper()              # Fast
df['email'].str.contains('@')       # Fast

# NOT this way (slow Python loop)
[df['name'].upper() for name in df['name']]  # Very slow

Compile Regex Patterns

import re

# Compile pattern once
pattern = re.compile(r'^[\w]+@')

# Use compiled pattern
df['email'].str.match(pattern)

# Better yet, use string methods when possible
df['email'].str.startswith('admin')  # Faster than regex

Avoiding Unnecessary Copies

In-Place Operations

# Create copy (uses extra memory)
df2 = df.copy()
df2['salary'] = df2['salary'] * 1.1

# Modify in place when possible
df['salary'] = df['salary'] * 1.1  # Same df, no copy

# Use inplace parameter where available (deprecated in future)
df.fillna(0, inplace=True)

Method Chaining

# Traditional (creates intermediate DataFrames)
df = pd.read_csv('file.csv')
df = df[df['salary'] > 80000]
df = df.dropna()
df = df.sort_values('salary')

# Method chaining (creates one DataFrame at the end)
df = (pd.read_csv('file.csv')
      .pipe(lambda x: x[x['salary'] > 80000])
      .dropna()
      .sort_values('salary'))

Assign for Method Chaining

# Using assign (clean method chaining)
df = (pd.read_csv('file.csv')
      .assign(
          bonus=lambda x: x['salary'] * 0.1,
          total=lambda x: x['salary'] + x['bonus']
      )
      .query('total > 80000')
      .sort_values('total'))

When to Use Alternatives

Pandas is not always the best tool. Know when to use alternatives.

Use Pandas When:

  • Data fits in RAM (typically < 5-10 GB)
  • Data is tabular (rows and columns)
  • You need interactive exploration
  • Prototyping and EDA

Consider Alternatives When:

ScenarioAlternative
Data > RAMDask, Vaex, PySpark
Distributed computingPySpark, Dask
Very large text dataPolars, DuckDB
Time series > RAMtsfresh, pyts
ML feature engineeringPolars, cuDF (GPU)

Polars as Alternative

# Polars is often faster than Pandas for:
# - Large datasets
# - Complex queries
# - Multi-threaded operations

import polars as pl

# Similar API to Pandas
df = pl.read_csv('file.csv')
df.filter(pl.col('salary') > 80000)
df.group_by('dept').agg(pl.col('salary').mean())
