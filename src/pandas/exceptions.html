Exceptions

Common Pandas errors, their causes, and how to fix them.

SettingWithCopyWarning

One of the most common warnings in Pandas. It occurs when you try to modify a view instead of a copy.

Common Causes

# Cause 1: Chained indexing
df[df['col'] > 0]['other_col'] = value  # Warning!

# Cause 2: Creating a view accidentally
subset = df[df['col'] > 0]
subset['other_col'] = value  # Warning affects 'subset' and potentially 'df'!

Solutions

# Solution 1: Use .loc for conditional assignment
df.loc[df['col'] > 0, 'other_col'] = value  # Correct!

# Solution 2: Make explicit copy
subset = df[df['col'] > 0].copy()
subset['other_col'] = value  # No warning

# Solution 3: Create DataFrame first, then assign
mask = df['col'] > 0
df.loc[mask, 'other_col'] = value
Warning: In Pandas 2.0+, Copy-on-Write is enabled by default, making this behavior more predictable. However, understanding the issue is still important for working with older code.

KeyError

Occurs when trying to access a column or index label that doesn't exist.

Common Causes

# Cause 1: Typo in column name
df['ColmnName']  # KeyError if column is 'ColumnName'

# Cause 2: Column doesn't exist
df['nonexistent']  # KeyError

# Cause 3: Wrong index access
df.loc['nonexistent_row']  # KeyError

# Cause 4: Case sensitivity
df['Name'] vs df['name']

Solutions

# Solution 1: Check if column exists first
if 'column' in df.columns:
    df['column']

# Solution 2: Use get method (returns None or default)
df.get('column', default_value)
df.get('column', 'default')

# Solution 3: Use safe access
df[['col1', 'col2']]  # Returns empty if none exist

# Solution 4: Rename column
df.rename(columns={'old': 'new'}, inplace=True)

ValueError

General error when an operation receives an argument with the right type but inappropriate value.

Shape Mismatch

# Cause: Assigning wrong shape
df = pd.DataFrame({'a': [1, 2, 3]})
df['b'] = [1, 2]  # ValueError: Length mismatch

# Solution: Match lengths
df['b'] = [1, 2, 3]  # Correct

# Or use compatible length
df['b'] = [1]  # Broadcasts to all rows

Cannot Merge Incompatible Keys

# Cause: Duplicate column names without suffix
df1 = pd.DataFrame({'key': [1, 2], 'val': [10, 20]})
df2 = pd.DataFrame({'key': [1, 2], 'val': [30, 40]})
pd.merge(df1, df2, on='key')  # ValueError

# Solution: Specify suffixes
pd.merge(df1, df2, on='key', suffixes=('_left', '_right'))

Cannot Convert Data

# Cause: Cannot convert string to numeric
pd.to_numeric(['1', '2', 'abc'])  # ValueError

# Solution: Use errors='coerce'
pd.to_numeric(['1', '2', 'abc'], errors='coerce')  # Returns [1, 2, NaN]

# Or filter first
pd.to_numeric(df['col'].dropna())

Invalid dtype Conversion

# Cause: Cannot convert
df['date'].astype('int64')  # ValueError if dates can't convert

# Solution: Convert to datetime first
df['date'] = pd.to_datetime(df['date'])
df['date'].astype('int64')  # Now works

TypeError

Occurs when an operation is applied to incompatible types.

String and Numeric Operations

# Cause: Cannot add string and number
df['col'] = 'text'
df['col'] + 5  # TypeError

# Solution: Convert first
df['col'].astype(int) + 5  # If numeric
pd.to_numeric(df['col'], errors='coerce') + 5  # Handle non-numeric

Boolean Indexing with Series

# Cause: Wrong boolean type
df[df['col']]  # TypeError if 'col' is not boolean

# Solution: Convert to boolean
df[df['col'].astype(bool)]

Operation on Non-numeric Data

# Cause: Sum on string column
df['name'].sum()  # TypeError

# Solution: Use appropriate string method
df['name'].str.join('')  # For concatenation

IndexError

Occurs when trying to access an index position that doesn't exist.

iloc Out of Bounds

# Cause: Index out of range
df = pd.DataFrame({'a': [1, 2, 3]})
df.iloc[10]  # IndexError

# Solution: Check bounds first
if len(df) > 10:
    df.iloc[10]

# Or use safe access
df.iloc[len(df) - 1]  # Last element

Empty Series Access

# Cause: Accessing empty series
s = pd.Series([])
s[0]  # IndexError

# Solution: Check if empty first
if len(s) > 0:
    s[0]

MergeError

Occurs during merge operations.

Duplicate Key Error

# Cause: Duplicate values on key column
pd.merge(df1, df2, on='key', validate='one_to_one')  # MergeError if duplicates

# Solution: Check for duplicates first
print(df1['key'].duplicated().sum())  # Count duplicates
print(df2['key'].duplicated().sum())

# Or use appropriate validation
pd.merge(df1, df2, on='key', validate='many_to_one')

No Common Columns

# Cause: No overlapping columns
df1 = pd.DataFrame({'a': [1, 2]})
df2 = pd.DataFrame({'b': [1, 2]})
pd.merge(df1, df2, on=['a', 'b'])  # MergeError

# Solution: Specify columns
pd.merge(df1, df2, left_on='a', right_on='b', how='cross')

ParserError

Occurs when reading files with malformed data.

CSV Parsing Issues

# Cause: Malformed CSV
pd.read_csv('file.csv')  # ParserError if file is malformed

# Solutions:
pd.read_csv('file.csv', on_bad_lines='skip')  # Skip bad lines
pd.read_csv('file.csv', error_bad_lines=False)  # Old syntax
pd.read_csv('file.csv', encoding='latin-1')  # Wrong encoding

# Check file first
with open('file.csv', 'r') as f:
    for i, line in enumerate(f):
        if i > 10:
            break
        print(line)

Date Parsing Issues

# Cause: Cannot parse dates
pd.to_datetime(['2024-01-01', 'invalid', '2024-01-03'])  # ParserError

# Solution: Use errors='coerce'
pd.to_datetime(['2024-01-01', 'invalid', '2024-01-03'], errors='coerce')

PermissionError / FileNotFoundError

File-related errors.

File Not Found

# Cause: Wrong path
pd.read_csv('wrong_path/file.csv')  # FileNotFoundError

# Solution: Use absolute path or check existence
import os
if os.path.exists('path/file.csv'):
    df = pd.read_csv('path/file.csv')
else:
    print("File not found")

Permission Denied

# Cause: File is open or no write permission
df.to_csv('file.csv')  # PermissionError

# Solution: Close file or use different path
df.to_csv('new_file.csv')
df.to_csv('file.xlsx', mode='a')  # Append if supported

Dtype Mismatch Errors

Object vs Numeric Comparison

# Cause: Comparing string to number
df[df['col'] > 5]  # UserWarning if 'col' is object type

# Solution: Convert column first
df['col'] = pd.to_numeric(df['col'], errors='coerce')
df[df['col'] > 5]

Merge Key Type Mismatch

# Cause: Different types in merge keys
df1['key'] = df1['key'].astype(str)
df2['key'] = df2['key'].astype(int)
pd.merge(df1, df2, on='key')  # May not match properly

# Solution: Convert to same type
df1['key'] = df1['key'].astype(int)

Performance and Memory Warnings

Memory Error

# Cause: Data too large for memory

# Solutions:
# 1. Read in chunks
chunks = pd.read_csv('large_file.csv', chunksize=100000)
df = pd.concat([chunk for chunk in chunks])

# 2. Reduce dtypes
df = pd.read_csv('file.csv', dtype={'col': 'int8'})

# 3. Use efficient formats
df = pd.read_parquet('file.parquet')

# 4. Drop unnecessary columns
df = pd.read_csv('file.csv', usecols=['needed', 'columns'])

Performance Warning

# Cause: Using slow operations on large data

# Solutions:
# 1. Use vectorized operations
df['col'] = df['col'] * 2  # Fast

# Instead of:
for i in range(len(df)):  # Very slow
    df.loc[i, 'col'] = df.loc[i, 'col'] * 2

# 2. Use appropriate dtypes
df['col'] = df['col'].astype('int8')  # Faster than int64

Common GroupBy Errors

KeyError in GroupBy

# Cause: Column doesn't exist
df.groupby('wrong_column')  # KeyError

# Solution: Check available columns
print(df.columns.tolist())

Specification Error

# Cause: Wrong named aggregation syntax
df.groupby('col').agg(new_name='other_col')  # Old syntax, wrong

# Solution: Use correct syntax
df.groupby('col').agg(new_name=('other_col', 'sum'))

Datetime Errors

Cannot Parse Date

# Cause: Wrong date format
pd.to_datetime('01/02/2024')  # Ambiguous (US vs EU)

# Solution: Specify format
pd.to_datetime('01/02/2024', format='%m/%d/%Y')  # US
pd.to_datetime('01/02/2024', format='%d/%m/%Y')  # EU

Timezone Issues

# Cause: Timezone-naive vs timezone-aware
df['time1'].dt.tz_localize('UTC')  # Add timezone
df['time1'].dt.tz_convert('US/Eastern')  # Convert timezone

Debugging Tips

Inspect DataFrame Before Operations

# Always check your data first
df.head()
df.dtypes
df.shape
df.info()

# Check for nulls
df.isna().sum()
df.isna().any()

Use try-except

try:
    df['col'] = pd.to_numeric(df['col'])
except ValueError as e:
    print(f"Cannot convert: {e}")
    # Handle error

Check dtypes Before Comparison

# Always check dtypes before comparisons
print(df['col'].dtype)
print(df['col'].dtype in ['int64', 'float64'])

# Convert if needed
df['col'] = pd.to_numeric(df['col'], errors='coerce')

Validate Merge Operations

# Always validate merges
pd.merge(df1, df2, on='key', validate='one_to_one')  # Raises error if not

Error Prevention Checklist

  • Always check df.dtypes before operations
  • Use df.info() to understand your data
  • Convert dtypes explicitly with pd.to_numeric() or astype()
  • Use .loc[] for conditional assignments
  • Check for nulls with df.isna().sum()
  • Validate merge operations with validate parameter
  • Handle file encoding explicitly with encoding parameter
  • Use errors='coerce' for problematic conversions
  • Check index types with df.index.dtype
  • Verify column existence with 'col' in df.columns
