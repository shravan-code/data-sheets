Common Spark Errors

OutOfMemoryError

PySparkSQLException: Exception of type 
'org.apache.spark.SparkException' with message 
'Job aborted due to stage failure: Task 0 in stage 1.0 
failed 4 times, most recent failure: 
Killed attempt... Loss was due to OutOfMemoryError'

Solutions:

  • Reduce partitions: df.coalesce(50)
  • Increase executor memory: spark.executor.memory
  • Use Kryo serialization
  • Filter data earlier to reduce memory usage
  • Use spark.sql.adaptive.enabled

Shuffle FetchFailedException

ShuffleFetchFailedException: 
Shuffle task 0 failed 4 times

Solutions:

  • Increase shuffle partitions: spark.sql.shuffle.partitions
  • Increase executor memory
  • Enable AQE: spark.sql.adaptive.enabled
  • Check for data skew

Python Worker Exited Unexpectedly

Python worker exited unexpectedly

Solutions:

  • Increase spark.python.worker.memory
  • Reduce data size per partition
  • Use Pandas UDF instead of regular UDF
  • Check for Python exceptions in UDF

Schema Mismatch

AnalysisException: cannot resolve given input columns

Solutions:

  • Check column names match exactly (case-sensitive)
  • Print schema: df.printSchema()
  • Use explicit schema instead of inference
  • Check for typos in column names

Column Name Conflicts

AnalysisException: Reference 'col' is ambiguous

Solutions:

  • Drop duplicate columns after join: df.drop(right_df.dept)
  • Rename before join: df.withColumnRenamed('col', 'new_col')
  • Use qualified names

Type Mismatch

TypeError: Unsupported type in UDF

Solutions:

  • Cast to correct type: df.withColumn('num', col('str').cast('int'))
  • Handle nulls in UDF
  • Check PySpark data types

Null Handling

# Null in comparison
df.filter(col('x') == None)  # WRONG
df.filter(col('x').isNull())  # CORRECT

# Null in aggregation
df.groupBy('dept').agg(F.sum('salary'))  # nulls handled

# Null in join
df1.join(df2, on='key', how='inner')  # nulls don't match

Divide by Zero

# Safe division
from pyspark.sql.functions import when, lit

df.withColumn('ratio', 
    when(col('denominator') == 0, lit(0))
    .otherwise(col('numerator') / col('denominator'))
)

Large Collect

# Avoid collect() on large data
df.collect()  # BRINGS ALL DATA TO DRIVER

# Better approaches
df.show()           # Print limited rows
df.take(100)        # Take first N rows
df.limit(100).collect()  # Limit then collect
df.write.parquet()  # Write directly to storage

Serialization Errors

# UDF can't serialize
def bad_udf(x):
    import local_module  # Can't access outside Spark
    return local_module.process(x)

# Use broadcast variables or wrap in UDF
from functools import partial

def good_udf(lookup_dict):
    @udf(returnType=StringType())
    def inner(x):
        return lookup_dict.get(x, 'unknown')
    return inner

lookup = {'a': 'A', 'b': 'B'}
broadcast_lookup = sc.broadcast(lookup)

Best Practices

  • Always define schemas explicitly for production code
  • Use try-except around actions
  • Log row counts at each stage
  • Use df.printSchema() to debug schema issues
  • Use df.explain() to debug performance
  • Test with small data first
