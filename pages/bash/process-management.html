Process Management

Background & Foreground Jobs

Processes can run in the background using & and be managed with jobs, fg, bg.

Job Control

sleep 100 &         # run in background
jobs                # list background jobs
fg %1               # bring job 1 to foreground
bg %1               # resume job 1 in background
kill %1             # kill job 1
wait                # wait for all background jobs

Running Multiple Tasks

task1 &
task2 &
task3 &
wait  # wait for all to finish
echo "All tasks complete"

Process Monitoring

Monitor running processes with ps, top, and pgrep.

Process Commands

ps aux                    # all processes
ps aux | grep nginx       # find specific process
pgrep nginx               # get PID of nginx
kill PID                  # terminate process
kill -9 PID               # force kill
pkill nginx               # kill by name
nohup ./longscript.sh &   # run ignoring hangup

Signals

Signals are notifications sent to processes. Use trap to catch signals in your scripts.

Trap Signals

cleanup() {
  echo "Cleaning up..."
  rm -f /tmp/tempfile.$$
  exit 0
}

trap cleanup INT TERM EXIT

echo "Script PID: $$"
sleep 100  # will clean up on Ctrl+C

Ignore SIGHUP

trap '' HUP   # ignore hangup signal
nohup ./script.sh &
