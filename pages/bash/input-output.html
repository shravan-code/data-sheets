4. Input & Output

The read command reads input from stdin. It can read into single or multiple variables.

Reading User Input


The read command reads input from stdin. It can read into single or multiple variables.


Basic Input


read -p "Enter your name: " name
echo "Hello, $name!"

Silent Input (Password)


read -s -p "Enter password: " password
echo
echo "Password entered (hidden)"

Read with Timeout


if read -t 5 -p "Enter within 5 seconds: " answer; then
  echo "You entered: $answer"
else
  echo "Timed out!"
fi

Read Array from Input


read -a items -p "Enter items separated by spaces: "
for item in "${items[@]}"; do
  echo "Item: $item"
done


Output with echo and printf


echo is simple; printf provides C-style formatted output with more control.


echo Options


echo "Simple output"
echo -n "No newline"    # no newline
echo -e "Tab:\there"    # interpret escapes
echo -e "\033[32mGreen Text\033[0m"  # colored

printf Formatting


printf "Name: %s, Age: %d\n" "Alice" 30
printf "Pi: %.4f\n" 3.14159
printf "%-15s %5d\n" "Item" 100  # left-aligned


Redirection


Redirect stdout, stderr, and stdin using >, >>, 2>, &>, and <.


Output Redirection


echo "Hello" > output.txt       # overwrite
echo "World" >> output.txt      # append
ls /nonexistent 2> error.txt    # stderr only
ls /nonexistent &> all.txt      # both stdout+stderr

Input Redirection


while read line; do
  echo "Line: $line"
done < input.txt

Discard Output


command > /dev/null 2>&1  # discard all output


Pipes


Pipes (|) connect stdout of one command to stdin of another, enabling powerful command chaining.


Basic Pipe


ls -la | grep ".sh" | sort

Count Lines with Pipe


cat /etc/passwd | wc -l

Process Substitution


diff <(ls dir1/) <(ls dir2/)  # compare two command outputs
