3. Data Types & Strings

Bash strings can be manipulated using parameter expansion for length, replacement, case conversion, and more.

String Operations


Bash strings can be manipulated using parameter expansion for length, replacement, case conversion, and more.


String Length


str="Hello World"
echo ${#str}  # 11

String Replace


msg="I like cats and cats"
echo ${msg/cats/dogs}   # Replace first: I like dogs and cats
echo ${msg//cats/dogs}  # Replace all:   I like dogs and dogs

Case Conversion (Bash 4+)


text="Hello World"
echo ${text^^}   # HELLO WORLD (uppercase)
echo ${text,,}   # hello world (lowercase)
echo ${text^}    # Hello World (capitalize first)

String Concatenation


first="Hello"
second="World"
combined="$first $second"
echo $combined  # Hello World


Arrays


Bash supports indexed and associative arrays. Indexed arrays use numeric indices; associative arrays use string keys.


Indexed Array


fruits=("apple" "banana" "cherry")
echo ${fruits[0]}    # apple
echo ${fruits[@]}    # all elements
echo ${#fruits[@]}   # 3 (length)
fruits+=("date")     # append
unset fruits[1]      # delete element

Array Iteration


colors=("red" "green" "blue")
for color in "${colors[@]}"; do
  echo "Color: $color"
done

Associative Array (Bash 4+)


declare -A person
person[name]="Alice"
person[age]=30
person[city]="NYC"
echo ${person[name]}        # Alice
echo ${!person[@]}          # keys: name age city
echo ${person[@]}           # values

Array Slicing


nums=(10 20 30 40 50)
echo ${nums[@]:1:3}   # 20 30 40 (offset 1, length 3)


Here Documents & Here Strings


Here-docs allow multi-line strings. Here-strings pass a single string to stdin.


Here Document


cat << EOF
Line 1
Line 2
Line 3
EOF

Here Document to File


cat << EOF > output.txt
Name: Alice
Age: 30
EOF

Here String


grep "hello" <<< "hello world"
# Output: hello world
