20. Cheatsheet: Bash vs PowerShell

Side-by-side comparison of common Bash and PowerShell commands and concepts.

Navigation & File System


BashPowerShell

cd /home/userSet-Location C:\Users\User
lsGet-ChildItem
ls -laGet-ChildItem -Force
ls -RGet-ChildItem -Recurse

Functions


BashPowerShell
greet() {
local name=$1
echo "Hello, $name!"
}
greet Alice
# --- POWERSHELL ---
function Greet {
param([string]$Name)
Write-Output "Hello, $Name!"
}
Greet -Name 'Alice'
# Return values
# BASH: use echo or $?
add() { echo $(( $1 + $2 )); }
result=$(add 3 4)
# POWERSHELL: return or implicit output
function Add { param($a,$b); return $a + $b }
$result = Add 3 4

Scripting Patterns


BashPowerShell
# Script args
$0, $1, $2$PSCommandPath, $args[0], $args[1]
$@$args
$#$args.Count
# Error handling
set -e$ErrorActionPreference = 'Stop'
command| echo 'failed' | try { cmd } catch { 'failed' }
$? == 0$? == $true
# String heredoc
cat <<EOF@"
texttext
EOF"@
# Command substitution
result=$(command)$result = command
result=$(echo hitr a-z A-Z) | $result = 'hi'.ToUpper()
# Null coalescing
${VAR:-default}$var ?? 'default' (PS7+)
# Exit codes
exit 0exit 0
exit 1exit 1
# Source another script
source ./functions.sh. .\functions.ps1 (dot-source)
. ./functions.shImport-Module .\MyModule.psm1
# Shebang line
#!/bin/bash# (no shebang; use #Requires -Version 7)
# Debug mode
bash -x script.shSet-PSDebug -Trace 2
set -xSet-PSDebug -Trace 1
set +xSet-PSDebug -Off
