PowerShell • Windows Automation

Remoting & Jobs

Execute commands on remote machines and run background jobs.

PowerShell Remoting

WS-Management (WinRM) based remoting allows running commands on remote computers.

POWERSHELL
# Enable remoting (run as admin)
Enable-PSRemoting -Force

# One-time remote command
Invoke-Command -ComputerName Server01 -ScriptBlock {
    Get-Process | Where-Object CPU -gt 10
}

# Persistent session
$session = New-PSSession -ComputerName Server01
Invoke-Command -Session $session -ScriptBlock { Get-Service }

# Interactive remote session
Enter-PSSession -ComputerName Server01
# ... work remotely ...
Exit-PSSession

# Run script file remotely
Invoke-Command -ComputerName Server01 -FilePath '.\script.ps1'

# Multiple computers
Invoke-Command -ComputerName Server01, Server02, Server03 -ScriptBlock {
    Get-WmiObject Win32_OperatingSystem | Select-Object PSComputerName, Caption, Version
}

# Copy files via remoting
Copy-Item 'C:\local.txt' -Destination 'C:\remote.txt' -ToSession $session

# Cleanup
Remove-PSSession $session

Background Jobs

Run tasks in the background without blocking the console.

POWERSHELL
# Start a background job
$job = Start-Job -ScriptBlock {
    Start-Sleep -Seconds 5
    Get-Process | Measure-Object | Select-Object -Exp Count
}

# Check job status
Get-Job
Get-Job -Id $job.Id

# Wait for completion
Wait-Job -Job $job

# Get results
Receive-Job -Job $job

# Remove completed jobs
Remove-Job -Job $job

# Named job
$namedJob = Start-Job -Name 'MyTask' -ScriptBlock { 'Done' }
Receive-Job -Name 'MyTask' -Wait -AutoRemoveJob

# Thread jobs (faster, no new process)
$threadJob = Start-ThreadJob -ScriptBlock { Get-Process }
Receive-Job $threadJob -Wait
