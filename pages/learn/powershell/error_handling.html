Error Handling

Managing errors gracefully using Try/Catch/Finally, error preferences, and custom errors.

Try / Catch / Finally

Structured exception handling for robust scripts.

POWERSHELL
try {
    $result = 10 / 0
    Get-Item 'C:\NonExistent\file.txt' -ErrorAction Stop
} catch [System.DivideByZeroException] {
    Write-Host 'Divide by zero error!'
} catch [System.IO.FileNotFoundException] {
    Write-Host 'File not found!'
} catch {
    Write-Host "Unexpected error: $($_.Exception.Message)"
    Write-Host "Type: $($_.Exception.GetType().FullName)"
} finally {
    Write-Host 'This always runs (cleanup here)'
}

# Error details
try {
    Get-Item 'C:\missing' -ErrorAction Stop
} catch {
    $_.Exception.Message
    $_.InvocationInfo.Line
    $_.ScriptStackTrace
}

ErrorAction & ErrorVariable

Control how cmdlets handle non-terminating errors.

POWERSHELL
# ErrorAction values: Stop, Continue, SilentlyContinue, Ignore, Inquire
Get-Item 'missing' -ErrorAction Stop            # Turns into terminating error
Get-Item 'missing' -ErrorAction SilentlyContinue # Suppresses error
Get-Item 'missing' -ErrorAction Ignore           # Ignores completely

# Capture errors in variable
Get-Item 'missing' -ErrorVariable myErrors -ErrorAction SilentlyContinue
if ($myErrors) {
    Write-Host "Error occurred: $($myErrors[0].Exception.Message)"
}

# Global preference
$ErrorActionPreference = 'Stop'     # All cmdlets throw
$ErrorActionPreference = 'Continue' # Default

# $Error automatic variable
$Error[0]   # Most recent error
$Error.Clear()

Throw & Custom Errors

Raise custom exceptions and write meaningful error records.

POWERSHELL
# Throw a string
function Validate-Age {
    param([int]$Age)
    if ($Age -lt 0 -or $Age -gt 150) {
        throw "Invalid age: $Age"
    }
    return $Age
}

# Throw an exception object
throw [System.ArgumentException]::new('Value cannot be empty', 'Name')

# Write non-terminating error
function Get-Data {
    param($Id)
    if (-not $Id) {
        $ex = [System.ArgumentNullException]::new('Id')
        $err = [System.Management.Automation.ErrorRecord]::new(
            $ex, 'InvalidId', 'InvalidArgument', $Id
        )
        $PSCmdlet.WriteError($err)
        return
    }
    return "Data for $Id"
}
