Networking

Network diagnostics, REST API calls, and network configuration.

Network Diagnostics

Built-in tools for testing connectivity and resolving DNS.

POWERSHELL
# Ping test
Test-Connection -ComputerName google.com
Test-Connection -ComputerName google.com -Count 4 -Quiet  # Returns True/False

# TCP port test
Test-NetConnection -ComputerName google.com -Port 443
(Test-NetConnection -ComputerName google.com -Port 80).TcpTestSucceeded

# DNS resolution
Resolve-DnsName google.com
Resolve-DnsName google.com -Type MX

# Network adapters
Get-NetAdapter
Get-NetIPAddress
Get-NetRoute

# Active connections
Get-NetTCPConnection -State Established

# Traceroute
Test-NetConnection -ComputerName google.com -TraceRoute

REST API Calls

Invoke-RestMethod and Invoke-WebRequest for HTTP calls.

POWERSHELL
# Simple GET
$response = Invoke-RestMethod -Uri 'https://jsonplaceholder.typicode.com/posts/1'
$response.title

# GET with headers
$headers = @{
    'Authorization' = 'Bearer mytoken'
    'Accept'        = 'application/json'
}
$data = Invoke-RestMethod -Uri 'https://api.example.com/data' -Headers $headers

# POST JSON
$body = @{
    name  = 'Alice'
    email = 'alice@example.com'
} | ConvertTo-Json

$result = Invoke-RestMethod `
    -Uri  'https://api.example.com/users' `
    -Method POST `
    -Body $body `
    -ContentType 'application/json' `
    -Headers $headers

# Download file
Invoke-WebRequest -Uri 'https://example.com/file.zip' -OutFile 'C:\Temp\file.zip'

# Web scraping
$page = Invoke-WebRequest -Uri 'https://example.com'
$page.Links | Select-Object href
$page.Content
