5. Stacks & Queues

LIFO vs FIFO, valid parentheses type problems.

Stacks & Queues

Both are data structures that hold collections of items, but they differ strictly in how items are added and removed.

Stack (LIFO)

Last In, First Out. Think of a stack of plates. You can only add a plate to the top (Push) and take a plate from the top (Pop).

Queue (FIFO)

First In, First Out. Think of a line at a coffee shop. You join the back of the line (Enqueue) and get served from the front (Dequeue).

flowchart TD subgraph Stack Push --> Top Top --> Pop end subgraph Queue Enqueue --> Back Front --> Dequeue end

What problem does it solve?

Stacks are used for tracking state that needs to be reversed or unwound, like the "Undo" button in text editors, browser history, and verifying matching brackets/parentheses.

Queues are used for scheduling and ordering tasks fairly, like printer jobs, handling web server requests, and Breadth-First Search (BFS).

Python Example: Valid Parentheses (Stack)

# Problem: Check if a string of brackets is valid
def is_valid_parentheses(s):
    stack = []
    mapping = {')': '(', '}': '{', ']': '['}
    
    for char in s:
        if char in mapping:
            # If stack is not empty, pop top element. Else use dummy '#'
            top_element = stack.pop() if stack else '#'
            if mapping[char] != top_element:
                return False
        else:
            # It's an opening bracket, push to stack
            stack.append(char)
            
    return not stack # True if stack is empty

print(is_valid_parentheses("{[()]}")) # Output: True
