Section 8: Joins

RIGHT JOIN

Base Table Context: employees

emp_id name dept_id
1 Alice 10
2 Bob 20
3 Carol 10
4 Dave 99

Base Table Context: departments

dept_id dept_name
10 Engineering
20 Marketing
30 HR

The RIGHT JOIN is used to combine rows from two or more tables based on a related column between them. This is essential for working with normalized databases where information is spread across multiple tables to reduce redundancy.

SELECT e.name, d.dept_name FROM employees e
RIGHT JOIN departments d ON e.dept_id = d.dept_id;
Output Result
name dept_name
Alice Engineering
Carol Engineering
Bob Marketing
NULL HR

HR dept included even though it has no employees.

Standard RIGHT JOIN implementation

SELECT 
  orders.order_id, 
  customers.customer_name 
FROM orders 
RIGHT JOIN customers 
  ON orders.customer_id = customers.customer_id;
Output
Returns combined rows where the join condition matches.
Pros / Advantages
  • Maintains data normalization
  • Allows complex relational mapping
  • Highly optimized by modern RDBMS
Cons / Limitations
  • Complexity increases with more tables
  • Cartesian products if join conditions are missing
Pro Tips & Tricks
  • Join on indexed columns (Primary/Foreign keys)
  • Filter data in WHERE before joining if possible

Troubleshooting & Common Issues

Issue: Duplicated Rows
Solution: Check for many-to-many relationships and use DISTINCT or specific filters.
