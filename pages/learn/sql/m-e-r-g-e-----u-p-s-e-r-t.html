Section 3: DML — Data Manipulation Language

MERGE / UPSERT

Base Table Context: employees

emp_id name dept_id salary
1 Alice 10 75000
2 Bob 20 60000
3 Carol 10 80000

Combines INSERT, UPDATE, and DELETE in a single statement based on whether a matching row exists. Also known as UPSERT.

PostgreSQL UPSERT with ON CONFLICT

INSERT INTO employees (emp_id, name, salary)
VALUES (1, 'Alice', 90000)
ON CONFLICT (emp_id)
DO UPDATE SET salary = EXCLUDED.salary;
Output
If emp_id=1 exists, updates salary. Otherwise inserts.

SQL Server / Standard SQL MERGE

MERGE INTO employees AS target
USING staging AS source
  ON target.emp_id = source.emp_id
WHEN MATCHED THEN
  UPDATE SET target.salary = source.salary
WHEN NOT MATCHED THEN
  INSERT (emp_id, name, salary) VALUES (source.emp_id, source.name, source.salary)
WHEN NOT MATCHED BY SOURCE THEN
  DELETE;
Output
Full sync: updates matches, inserts new, deletes removed rows.
Pros / Advantages
  • Granular control over data
  • Transactional safety (ACID)
Cons / Limitations
  • Generates transaction logs (WAL)
  • Can cause row-level locking
Pro Tips & Tricks
  • Batch your operations to reduce commit overhead
  • Use WHERE clauses carefully to avoid accidental mass updates

Troubleshooting & Common Issues

Issue: Slow Execution
Solution: Ensure columns in WHERE clause are indexed.
