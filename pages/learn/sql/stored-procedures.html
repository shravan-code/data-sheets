Section 21: Stored Procedures & Functions

Stored Procedures

Pre-compiled SQL code stored in the database. Can contain logic, multiple statements, and transaction control. Cannot be used inside SELECT.

Syntax Pattern

-- PostgreSQL:
CREATE OR REPLACE PROCEDURE give_raise(dept INT, pct NUMERIC)
LANGUAGE plpgsql AS $$
BEGIN
  UPDATE employees
  SET salary = salary * (1 + pct/100)
  WHERE dept_id = dept;
  COMMIT;
END;
$$;

-- Execute:
CALL give_raise(10, 10);
Output
Execution result varies based on data.

Basic structure of a Stored Procedures

-- Note: Syntax varies by RDBMS (Postgres/MySQL/SQL Server)
CREATE STORED PROCEDURES example_object() 
AS 
BEGIN 
  -- Logic here 
END;
Output
Stored Procedures compiled and stored in the database.
Pro Tips & Tricks
  • Always verify your logic with a small dataset first
  • Use EXPLAIN to see how the database handles your query
