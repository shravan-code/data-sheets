Section 18: Indexes

Partial Index

An INDEX is a powerful database object that improves the speed of data retrieval operations. It works like a book's index, allowing the database engine to find rows without scanning the entire table, though it comes with a storage and write-performance cost.

Syntax Pattern

-- Index only active employees:
CREATE INDEX idx_active_employees ON employees(name) WHERE status = 'active';
-- Only useful when query also has: WHERE status = 'active'
Output
Execution result varies based on data.

Creating a Partial Index

CREATE PARTIAL INDEX idx_customer_email 
ON customers (email);
Output
Index created successfully. Read performance on 'email' will improve.
Pros / Advantages
  • Dramatic speedup for SELECT queries
  • Enforces uniqueness
  • Reduces I/O overhead
Cons / Limitations
  • Slows down writes (INSERT/UPDATE/DELETE)
  • Takes up additional disk space
Pro Tips & Tricks
  • Only index columns used in WHERE, JOIN, or ORDER BY
  • Avoid indexing columns with very low cardinality (e.g. Booleans)

Troubleshooting & Common Issues

Issue: Index Not Used
Solution: Check EXPLAIN plan; functions on indexed columns (e.g. WHERE UPPER(name)) prevent index usage.
