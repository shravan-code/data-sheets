Section 29: Top 50 Conceptual Interview Questions

Top 50 Conceptual Interview Questions

1 What is the difference between WHERE and HAVING?

A: WHERE filters individual rows BEFORE aggregation (GROUP BY). HAVING filters groups AFTER aggregation. You cannot use aggregate functions in WHERE. Example: WHERE salary > 50000 filters rows; HAVING COUNT(*) > 2 filters groups.

2 What is a primary key vs a foreign key?

A: PRIMARY KEY uniquely identifies each row in a table and cannot be NULL. FOREIGN KEY is a column that references the PRIMARY KEY of another table, enforcing referential integrity between tables.

3 What is normalization? What are 1NF, 2NF, and 3NF?

A: Normalization organizes data to reduce redundancy. 1NF: atomic values, no repeating groups. 2NF: 1NF + no partial dependencies (all non-key columns depend on the full primary key). 3NF: 2NF + no transitive dependencies (non-key columns don't depend on other non-key columns).

4 What is the difference between INNER JOIN and OUTER JOIN?

A: INNER JOIN returns only matched rows from both tables. OUTER JOIN returns all rows from one or both tables, with NULLs where there's no match. LEFT OUTER JOIN: all rows from left + matched from right. RIGHT: vice versa. FULL: all rows from both.

5 What is ACID? Explain each property.

A: Atomicity: all or nothing — if part fails, all rolls back. Consistency: data moves between valid states. Isolation: concurrent transactions don't interfere. Durability: committed data persists through failures.

6 What is the difference between DELETE, TRUNCATE, and DROP?

A: DELETE removes specific rows (logged, can be rolled back, WHERE clause supported). TRUNCATE removes all rows fast (minimal logging, cannot filter, resets sequences). DROP removes the entire table structure and data permanently.

7 What are window functions and how do they differ from GROUP BY?

A: Window functions compute values across related rows without collapsing them into groups. GROUP BY collapses rows into one per group — you lose individual row detail. Window functions keep all rows while adding computed columns (running totals, ranks, etc.).

8 What is a CTE and when would you use it over a subquery?

A: A CTE (WITH clause) is a named temporary result set. Use CTE over subquery when: the result is needed multiple times in the same query, for recursive queries, or to improve readability. CTEs are not inherently faster — they're a readability/organizational tool.

9 Explain the difference between RANK, DENSE_RANK, and ROW_NUMBER.

A: ROW_NUMBER: unique sequential number (no ties). RANK: same rank for ties but skips the next rank (1,1,3). DENSE_RANK: same rank for ties but no gap (1,1,2).

10 What is an index and how does it improve query performance?

A: An index is a data structure (typically B-tree) that stores ordered pointers to rows. Instead of scanning the whole table (O(n)), the DB can navigate the B-tree (O(log n)) to find rows quickly. Trade-off: faster reads, slower writes (index must be maintained on INSERT/UPDATE/DELETE).

11 What is the difference between a clustered and non-clustered index?

A: A clustered index determines the physical storage order of the table — one per table (in SQL Server it's the PRIMARY KEY). A non-clustered index is a separate structure with pointers back to the table. PostgreSQL uses heap storage (no clustered index by default, but CLUSTER command exists).

12 What is a covering index?

A: An index that includes all columns needed by a query, so the query can be satisfied purely from the index without accessing the table data (index-only scan). Achieved using INCLUDE columns in PostgreSQL and SQL Server.

13 What is the difference between a view and a materialized view?

A: A regular view is a stored SQL query that executes on every use — no physical data stored. A materialized view stores the query result physically. Materialized views are faster for reads but must be refreshed to reflect updated data.

14 What are transaction isolation levels? Name them in order.

A: From lowest to highest isolation: Read Uncommitted, Read Committed, Repeatable Read, Serializable. Higher isolation prevents more anomalies (dirty reads, non-repeatable reads, phantom reads) but reduces concurrency.

15 What is a deadlock and how can it be prevented?

A: A deadlock occurs when two transactions each hold a lock the other needs, causing them to wait forever. Prevention: always acquire locks in the same order across transactions, use shorter transactions, use timeout parameters, or use SKIP LOCKED.

16 What is MVCC?

A: Multi-Version Concurrency Control maintains multiple versions of rows simultaneously. Readers see a consistent snapshot of data without blocking writers, and writers don't block readers. PostgreSQL uses MVCC extensively. Old row versions are cleaned up by VACUUM.

17 What is the difference between UNION and UNION ALL?

A: UNION combines two result sets and removes duplicates (requires sorting/hashing). UNION ALL combines without removing duplicates and is faster. Use UNION ALL unless you specifically need duplicate removal.

18 What is the N+1 problem?

A: Making 1 query to fetch N records, then N additional queries to fetch related data for each. Total: N+1 queries. Solution: use JOINs or batch loading to fetch all data in fewer queries.

19 What is the difference between correlated and non-correlated subqueries?

A: A non-correlated subquery executes independently once and returns a result used by the outer query. A correlated subquery references columns from the outer query and is executed once per row of the outer query — can be slow on large tables.

20 What is denormalization and when would you use it?

A: Denormalization intentionally adds redundancy to improve read performance by reducing JOINs. Common in data warehouses where reads outnumber writes. Example: storing dept_name in the employees table to avoid joining departments every query.

21 What is the difference between a Star Schema and a Snowflake Schema?

A: Star schema: fact table at center with denormalized dimension tables (fewer JOINs, simpler queries, more storage). Snowflake schema: dimension tables normalized into sub-dimensions (less redundancy, more JOINs, complex queries). Star is preferred for analytics.

22 What is SCD Type 2?

A: Slowly Changing Dimension Type 2 tracks historical changes by inserting a new row for each change with start/end dates and an is_current flag. The old row is expired (end_date set). This preserves full history. Common for customer/product dimensions in data warehouses.

23 How does SQL execution order differ from written order?

A: Written: SELECT, FROM, WHERE, GROUP BY, HAVING, ORDER BY. Executed: FROM → JOIN → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → LIMIT. This is why you cannot use a SELECT alias in a WHERE clause — WHERE runs before SELECT.

24 What is the difference between OLTP and OLAP?

A: OLTP (Online Transaction Processing): optimized for frequent, small, fast read/write operations. Highly normalized. Examples: transactional app databases. OLAP (Online Analytical Processing): optimized for complex analytical queries on large datasets. Denormalized, columnar. Examples: data warehouses.

25 What is a surrogate key vs natural key?

A: Natural key: a real-world attribute that uniquely identifies a record (e.g., SSN, email). Surrogate key: a system-generated identifier (e.g., SERIAL/AUTO_INCREMENT integer or UUID). Surrogate keys are preferred in data warehouses because natural keys can change and may expose PII.

26 What is a self-join and when would you use it?

A: A self-join is a join of a table to itself using aliases. Common for hierarchical data (org charts, category trees) where a row references another row in the same table via a manager_id or parent_id column.

27 What is the difference between a stored procedure and a function?

A: A function returns a value (scalar or table) and can be used in SELECT. A stored procedure does not return a value (or uses OUT params), cannot be used in SELECT, but can commit/rollback transactions.

28 What is a trigger?

A: A trigger is a stored procedure that automatically fires in response to a DML event (INSERT, UPDATE, DELETE) on a table. Used for auditing, enforcing constraints, or propagating changes. Can be BEFORE or AFTER the event, and FOR EACH ROW or FOR EACH STATEMENT.

29 What is the difference between optimistic and pessimistic locking?

A: Pessimistic locking locks the row when reading (SELECT FOR UPDATE), assuming a conflict will occur. Optimistic locking doesn't lock but uses a version column — at write time, checks that the version hasn't changed since reading. Pessimistic is safer under high contention; optimistic is more scalable under low contention.

30 What is partition pruning?

A: When querying a partitioned table, partition pruning is the optimizer skipping partitions that cannot contain the data matching the query predicate. For example, querying WHERE sale_date = '2024-06-01' on a date-partitioned table only scans the June 2024 partition.

31 What is a recursive CTE and when do you use it?

A: A recursive CTE references itself in its definition and consists of an anchor (base case) and a recursive member. Used for hierarchical or graph data: org charts, file systems, bill of materials, finding paths in a graph.

32 What is the purpose of the EXPLAIN ANALYZE command?

A: EXPLAIN shows the query planner's estimated execution plan without executing. EXPLAIN ANALYZE actually executes the query and shows both estimated and actual rows/times per node. Used to identify bottlenecks: Seq Scans, expensive sorts, bad row estimates, slow joins.

33 What is an idempotent query and why is it important in data engineering?

A: An idempotent query produces the same result no matter how many times it is run. Critical in data pipelines because failures and retries are common — non-idempotent pipelines cause duplicate or incorrect data. Achieved with UPSERT, DELETE+INSERT, or TRUNCATE+INSERT patterns.

34 What is CDC (Change Data Capture)?

A: CDC captures row-level changes (inserts, updates, deletes) as they happen in a source system and streams or logs them for downstream consumption. Approaches: timestamp-based, log-based (Debezium reads DB transaction logs), trigger-based. Used in real-time data pipelines and event streaming.

35 What is the difference between COALESCE and ISNULL/NVL?

A: COALESCE is ANSI SQL standard and accepts multiple arguments — returns the first non-NULL. ISNULL (SQL Server) and NVL (Oracle) take only 2 arguments. COALESCE is preferred for portability: COALESCE(a, b, c) vs ISNULL(a, ISNULL(b, c)).

36 What is the difference between CHAR and VARCHAR?

A: CHAR(n) is fixed-length — always stores exactly n bytes, padding with spaces. VARCHAR(n) is variable-length — stores only the actual length, up to n bytes. CHAR is slightly faster for fixed-length data (e.g., country codes); VARCHAR saves space for variable-length strings.

37 What is a lateral join?

A: LATERAL (in PostgreSQL) or CROSS APPLY/OUTER APPLY (in SQL Server) allows a subquery in the FROM clause to reference columns from preceding tables. It runs once per row, like a correlated subquery but in FROM. Useful for top-N per group or applying functions that return tables.

38 What is NULLIF and what problem does it solve?

A: NULLIF(a, b) returns NULL if a equals b, otherwise returns a. Most common use: prevent division by zero — NULLIF(denominator, 0) returns NULL instead of throwing an error when denominator is 0.

39 Explain the difference between ROWS and RANGE in window functions.

A: ROWS defines the window frame by physical row position (row count). RANGE defines it by the logical value range of the ORDER BY column. Example: with duplicates, ROWS BETWEEN 1 PRECEDING AND CURRENT ROW always includes exactly 2 rows; RANGE with duplicates may include more rows with the same ORDER BY value.

40 What is query rewriting and how does the optimizer use it?

A: The query optimizer internally rewrites queries to more efficient equivalent forms: converting NOT IN to NOT EXISTS, pushing predicates into subqueries, reordering joins, flattening subqueries into joins. Understanding this helps write queries the optimizer can rewrite effectively.

41 How do you find duplicates in a table?

A: Use GROUP BY with HAVING COUNT > 1: SELECT email, COUNT(*) FROM users GROUP BY email HAVING COUNT(*) > 1. To delete duplicates keeping one row, use ROW_NUMBER() to assign ranks and delete rows where rank > 1.

42 What is a full text search and how does it differ from LIKE?

A: LIKE does simple pattern matching on characters — can't rank by relevance, doesn't understand language. Full-text search tokenizes text, builds an inverted index, supports relevance ranking, stemming, stop words, and complex queries. Postgres uses tsvector/tsquery; MySQL has FULLTEXT indexes.

43 What is the difference between ELT and ETL?

A: ETL (Extract, Transform, Load): data is transformed before loading into the warehouse. ELT (Extract, Load, Transform): raw data is loaded first, then transformed inside the warehouse using SQL. ELT is common with cloud warehouses (BigQuery, Snowflake) where compute is powerful and cheap.

44 What are GROUPING SETS, ROLLUP, and CUBE?

A: All are extensions of GROUP BY for multi-dimensional aggregation. ROLLUP: hierarchical subtotals + grand total. CUBE: all possible combinations of group by columns. GROUPING SETS: explicit list of grouping combinations — most flexible.

45 What is a composite index and what is the leftmost prefix rule?

A: A composite index covers multiple columns in order. The leftmost prefix rule states that a query must include the leftmost column(s) of the index to benefit from it. Index on (dept_id, salary): WHERE dept_id = 10 AND salary > 5000 uses it; WHERE salary > 5000 does not.

46 Why does using a function on an indexed column prevent index usage?

A: When you wrap an indexed column in a function (WHERE YEAR(hire_date) = 2024), the database must compute the function for every row and cannot use the index on hire_date directly. Solution: rewrite the condition to reference the column directly: WHERE hire_date BETWEEN '2024-01-01' AND '2024-12-31'.

47 What is the difference between a clustered and non-clustered index in SQL Server?

A: SQL Server clustered index: physically sorts the table data in key order — one per table, usually the primary key. Non-clustered: separate B-tree with leaf nodes containing pointers to the clustered key (or RID for heap tables). PostgreSQL always uses heaps; no native clustered index.

48 What are the differences between EXISTS and IN?

A: IN evaluates the subquery and compares each row against the list — can be slow if the subquery returns many rows. EXISTS just checks whether any row matches — stops after the first match, generally faster for large subquery results. NOT EXISTS is often faster than NOT IN (which returns no results if any NULL exists in the subquery).

49 What is the purpose of VACUUM in PostgreSQL?

A: PostgreSQL's MVCC leaves behind dead row versions after UPDATE/DELETE. VACUUM reclaims the storage used by dead tuples and updates statistics used by the planner. VACUUM ANALYZE does both. AUTOVACUUM runs automatically but manual VACUUM is needed for heavily updated tables.

50 What is the difference between SAVEPOINT and ROLLBACK?

A: ROLLBACK undoes the entire current transaction back to the BEGIN. SAVEPOINT creates a named checkpoint within a transaction — ROLLBACK TO SAVEPOINT allows partial rollback to that point while keeping the transaction open. Useful when you want to undo only part of a transaction.
