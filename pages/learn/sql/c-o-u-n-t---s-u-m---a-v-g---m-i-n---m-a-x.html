Section 9: Aggregations & Grouping

COUNT, SUM, AVG, MIN, MAX

Base Table Context: sales

sale_id rep region amount product
1 Alice East 5000 Widget
2 Bob West 3000 Gadget
3 Alice East 7000 Gadget
4 Carol East 4000 Widget
5 Bob West 6000 Widget
6 Carol West 2000 Gadget

Aggregate functions collapse multiple rows into a single value.

Basic aggregates

SELECT
  COUNT(*)        AS total_sales,
  SUM(amount)     AS total_revenue,
  AVG(amount)     AS avg_sale,
  MIN(amount)     AS min_sale,
  MAX(amount)     AS max_sale
FROM sales;
Output Result
total_sales total_revenue avg_sale min_sale max_sale
6 27000 4500 2000 7000

COUNT DISTINCT

SELECT COUNT(DISTINCT rep) AS unique_reps FROM sales;
Output Result
unique_reps
3
Pros / Advantages
  • Powerful data summarization
  • Native performance for large datasets
Cons / Limitations
  • Can be memory intensive
  • Requires careful handling of NULLs
Pro Tips & Tricks
  • Use COUNT(1) or COUNT(*) instead of COUNT(column) if you don't need to skip NULLs
  • Filter rows using WHERE before grouping to reduce work

Troubleshooting & Common Issues

Issue: Missing Rows
Solution: Remember that Aggregate functions (except COUNT) ignore NULLs.
