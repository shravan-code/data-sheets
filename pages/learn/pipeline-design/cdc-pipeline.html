Schema Registry Evolution

How to handle source DB changes without breaking the stream.

  1. Forward Compatibility: Older consumers can read newer data.
  2. Backward Compatibility: Newer consumers can read older data.
  3. Full Compatibility: Both are true.

Pro-Tip: Log Compaction

Enable cleanup.policy=compact on your Kafka CDC topics. This keeps only the Latest State for every primary key, saving space while allowing full state recovery.
