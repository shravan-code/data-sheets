13. Security in Pipelines

Encryption, PII masking, and secrets management.

Security Architecture

graph TD S[Secrets Manager] --> P[Pipeline Engine] K[KMS Master Key] --> P P -- Encrypt --> D[Data Lake S3] D -- Masking Policy --> U[End User]

Implementing RBAC

Granting Least Privilege. Your ingestion service should have WRITE access to Bronze but NO access to Gold.

PII Masking at Scale

-- Snowflake Masking Policy
CREATE MASKING POLICY email_mask AS (val string) 
  RETURNS string ->
  CASE 
    WHEN current_role() IN ('ADMIN') THEN val
    ELSE '***@***.com'
  END;

Common Pitfall: Secrets in Logs

Never print the pipeline configuration to stdout. It might contain connection strings or API keys. Use a logger that Redacts sensitive strings.
