Decorators

Basic Decorator

A decorator wraps a function to add behavior before/after it.

import functools

def timer(func):
    @functools.wraps(func)  # preserve metadata
    def wrapper(*args, **kwargs):
        import time
        start = time.perf_counter()
        result = func(*args, **kwargs)
        end = time.perf_counter()
        print(f'{func.__name__} took {end-start:.6f}s')
        return result
    return wrapper

@timer
def slow_add(a, b):
    return a + b

result = slow_add(3, 4)
print(result)
print(slow_add.__name__)  # 'slow_add' (preserved by wraps)
Output
slow_add took 0.000002s
7
slow_add

Decorator with Arguments

Add a layer of wrapping to accept decorator arguments.

def repeat(n):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(3)
def say(msg):
    print(msg)

say('Hello!')
Output
Hello!
Hello!
Hello!

Stacked Decorators

Decorators are applied bottom-up.

def bold(func):
    @functools.wraps(func)
    def wrapper(*a, **kw):
        return '<b>' + func(*a, **kw) + '</b>'
    return wrapper

def italic(func):
    @functools.wraps(func)
    def wrapper(*a, **kw):
        return '<i>' + func(*a, **kw) + '</i>'
    return wrapper

@bold       # applied second
@italic     # applied first
def text(s):
    return s

print(text('Hello'))   # <b><i>Hello</i></b>

Class-based Decorator

Use a class with __call__ as a decorator.

class retry:
    def __init__(self, max_attempts=3):
        self.max_attempts = max_attempts
    
    def __call__(self, func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(self.max_attempts):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    print(f'Attempt {attempt+1} failed: {e}')
            raise RuntimeError('All attempts failed')
        return wrapper

@retry(max_attempts=3)
def flaky_function():
    import random
    if random.random() < 0.7:
        raise ValueError('Random failure')
    return 'Success!'

# try running (may succeed or exhaust retries)
Output
Attempt 1 failed: Random failure
Attempt 2 failed: Random failure
'Success!' (or RuntimeError)

Interview Mastery

Q:

Why use @functools.wraps in decorators?

A:

Without it, the wrapper function replaces the original, losing __name__, __doc__, __module__, etc. This breaks introspection, documentation tools, and debugging. @wraps copies these attributes from the wrapped function to the wrapper.

Q:

What is the difference between a decorator and a context manager?

A:

Decorators modify or wrap a function's behavior permanently. Context managers manage resources for a block of code using with statement — they guarantee setup/teardown (like file closing). Both can be implemented similarly but serve different use cases.
