Section 1: Foundations

Strings — All Methods

Strings in Python are immutable sequences of Unicode characters. They have a rich set of methods for manipulation.

String Creation & Slicing

Strings can be created with single, double, or triple quotes. Slicing uses [start:stop:step].

s = 'Hello, World!'
print(s[0])       # H
print(s[-1])      # !
print(s[0:5])     # Hello
print(s[7:])      # World!
print(s[:5])      # Hello
print(s[::2])     # Hlo ol!
print(s[::-1])    # !dlroW ,olleH (reversed)
Output
H
!
Hello
World!
Hello
Hlo ol!
!dlroW ,olleH

Case Methods: upper & lower

Convert the entire string to uppercase or lowercase. Useful for case-insensitive comparisons or normalizing user input.

s = 'hello WORLD'
print(s.upper())        # HELLO WORLD
print(s.lower())        # hello world

Case Methods: capitalize & title

capitalize() uppers only the first character; title() uppers the first character of each word. Perfect for formatting names or headings.

s = 'hello world'
print(s.capitalize())   # Hello world
print(s.title())        # Hello World

Search: find & index

Locate a substring. find() returns -1 if missing, whereas index() raises an error. Use find() when absence is expected, and index() when it should be present.

s = 'hello world'
print(s.find('world'))   # 6
print(s.find('xyz'))     # -1
print(s.index('world'))  # 6

Search: count

Count non-overlapping occurrences of a substring. Useful for basic frequency analysis.

s = 'banana'
print(s.count('a'))      # 3
print(s.count('na'))     # 2

Search: startswith & endswith

Check if a string starts or ends with a specific prefix/suffix. Faster and more readable than manual slicing.

filename = 'data.csv'
print(filename.startswith('data')) # True
print(filename.endswith('.csv'))   # True

Check Methods: isalpha & isdigit

Validate if string contains only letters or only digits. Critical for input validation (e.g., age vs name).

print('Python'.isalpha()) # True
print('12345'.isdigit())  # True
print('Python3'.isalpha()) # False (has digit)
Output
True
True
False

Modify: strip, lstrip, rstrip

Remove leading/trailing whitespace or specific characters. Essential for cleaning data from files or user input.

s = '  clean me  '
print(repr(s.strip()))    # 'clean me'
print('***hi***'.strip('*')) # hi

Modify: replace

Replace all occurrences of a substring with another. Useful for template filling or correcting data.

s = 'hello world'
print(s.replace('world', 'Python')) # hello Python
print('1,2,3'.replace(',', ';'))    # 1;2;3

Split & Join

split() converts a string into a list; join() merges a list back into a string. The foundation of text processing in Python.

s = 'apple,banana,cherry'
lst = s.split(',')
print(lst)                # ['apple', 'banana', 'cherry']

# join
print(' -> '.join(lst))   # apple -> banana -> cherry

Alignment: center, ljust, rjust

Pad strings with spaces or characters to a fixed width. Essential for creating aligned tables or reports in text-based interfaces.

s = 'hi'
print(s.center(10))        #     hi    
print(s.ljust(10, '.'))    # hi........
print(s.rjust(10, '.'))    # ........hi

Alignment: zfill

Pad numeric strings with leading zeros. Primarily used for formatting IDs, serial numbers, or fixed-width numerical data.

print('42'.zfill(5))       # 00042
print('-42'.zfill(5))      # -0042

String Formatting: f-strings

The modern and fastest way to embed expressions in string literals. Introduced in Python 3.6.

name, price = 'Widget', 19.99
print(f'{name} costs ${price:.2f}')
print(f'2 + 2 = {2 + 2}')
Output
Widget costs $19.99
2 + 2 = 4

String Formatting: .format()

A powerful formatting method that preceded f-strings. Still useful for reusable templates.

tpl = 'ID: {id}, Status: {st}'
print(tpl.format(id=101, st='Active'))
Output
ID: 101, Status: Active

Partitioning

split into exactly 3 parts: (before, separator, after). Safer than split() when you only need to divide at the first occurrence.

s = 'user@example.com'
print(s.partition('@'))    # ('user', '@', 'example.com')

Translate & Maketrans

Efficiently map multiple characters at once. Perfect for complex character replacements or deletions.

table = str.maketrans('aeiou', '12345')
print('hello world'.translate(table))
Output
h2ll4 w4rld

Encode/Decode

Convert between str and bytes.

s = 'hello'
b = s.encode('utf-8')      # str -> bytes
print(b)                   # b'hello'
print(type(b))             # bytes

decoded = b.decode('utf-8')  # bytes -> str
print(decoded)               # hello

# Unicode example
u = 'caf\u00e9'  # café
print(u.encode('utf-8'))   # b'caf\xc3\xa9'
Output
b'hello'
<class 'bytes'>
hello
b'caf\xc3\xa9'

Interview Mastery

Q: Why is string concatenation in a loop bad? What's the alternative?

A: Strings are immutable, so each `s += x` creates a new string object — O(n²) total. Use `''.join(list_of_strings)` which is O(n). Example: `result = ''.join(str(i) for i in range(1000))`.

Q: What's the difference between find() and index()?

A: `find()` returns -1 if substring not found. `index()` raises a `ValueError`. Use `find()` when absence is a valid case; use `index()` when absence means a bug.

Q: How do f-strings differ from .format()?

A: f-strings (3.6+) are evaluated at runtime and generally faster. They support expressions inline: `f'{2+2}'`. `.format()` is more flexible for templates stored as strings since f-strings must be defined inline. f-strings also support `=` for debugging: `f'{var=}'`.
