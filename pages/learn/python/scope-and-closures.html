Scope & Closures

The LEGB Rule

Python's name resolution order: Local -> Enclosing -> Global -> Built-in. Understanding this is key to avoiding 'undefined variable' errors.

x = 'global'
def outer():
    x = 'enclosing'
    def inner():
        x = 'local'
        print(f'Inner: {x}')
    inner()
outer()
Output
Inner: local

Managing State: global & nonlocal

Modify variables outside the local scope. global targets the module level; nonlocal targets the nearest enclosing scope (useful for stateful closures).

count = 0
def increment():
    global count
    count += 1

def outer():
    x = 0
    def inner():
        nonlocal x
        x += 1
        return x
    return inner

c = outer()
print(c(), c())
Output
1 2

Closures: Data Hiding

A function that remembers variables from its enclosing scope even after that scope has finished executing. Perfect for data hiding or factory patterns.

def multiplier(n):
    def multiply(x):
        return x * n
    return multiply

double = multiplier(2)
print(double(5))
Output
10

Interview Mastery

Q:

Explain the closure variable capture gotcha in loops.

A:

Functions in a loop capture the variable, not its value: `funcs = [lambda: i for i in range(3)]` — all return 2 (last value). Fix: use default arg: `lambda i=i: i` or `functools.partial`.
