Common Python Optimizations

Key patterns for writing faster Python code.

import timeit
from functools import lru_cache

# 1. Local variable lookup is faster than global/attribute
def slow_loop(n):
    result = []
    append = result.append  # cache method lookup!
    for i in range(n):
        append(i)
    return result

# 2. Use built-ins (implemented in C)
nums = list(range(1000))

# Slow: Python loop
def py_sum(lst):
    total = 0
    for x in lst:
        total += x
    return total

# Fast: built-in
t1 = timeit.timeit(lambda: py_sum(nums), number=10000)
t2 = timeit.timeit(lambda: sum(nums), number=10000)
print(f'loop: {t1:.3f}s, sum(): {t2:.3f}s')

# 3. String join over concatenation
def bad_concat(parts):
    s = ''
    for p in parts:
        s += p  # O(n^2) - new object each time
    return s

def good_join(parts):
    return ''.join(parts)  # O(n)

parts = ['word'] * 1000
t1 = timeit.timeit(lambda: bad_concat(parts), number=1000)
t2 = timeit.timeit(lambda: good_join(parts), number=1000)
print(f'concat: {t1:.3f}s, join: {t2:.3f}s')

# 4. Memoization with lru_cache
@lru_cache(maxsize=None)
def fib(n):
    if n < 2: return n
    return fib(n-1) + fib(n-2)

print(fib(50))  # instant

# 5. Use sets for membership testing O(1) vs list O(n)
nums_list = list(range(10000))
nums_set  = set(range(10000))

t1 = timeit.timeit(lambda: 9999 in nums_list, number=100000)
t2 = timeit.timeit(lambda: 9999 in nums_set,  number=100000)
print(f'list: {t1:.3f}s, set: {t2:.3f}s')

Interview Mastery

Q:

What are the most impactful Python performance optimizations?

A:

1) Use built-in functions (sum, map, filter) over loops — they're C-implemented. 2) Use sets for O(1) membership testing. 3) Use ''.join() not string concatenation. 4) Cache expensive results with lru_cache. 5) Use generators for large data. 6) Profile first — never guess. 7) For numeric heavy-lifting, use NumPy vectorization.
