Logging

Logging Module Complete

Use logging instead of print for production code. Five levels: DEBUG < INFO < WARNING < ERROR < CRITICAL.

import logging

# Basic config
logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)

logger = logging.getLogger(__name__)

logger.debug('Debug detail')        # development only
logger.info('Service started')      # normal events
logger.warning('Low disk space')    # something unexpected
logger.error('Failed to connect')   # error occurred
logger.critical('System down!')     # serious failure

# Log with exception info
try:
    1/0
except ZeroDivisionError:
    logger.exception('Caught an error')  # includes traceback

Handlers and Formatters

Route logs to different destinations with different formats.

import logging

logger = logging.getLogger('myapp')
logger.setLevel(logging.DEBUG)

# Console handler (INFO+)
console = logging.StreamHandler()
console.setLevel(logging.INFO)
console.setFormatter(logging.Formatter('%(levelname)s: %(message)s'))

# File handler (DEBUG+)
file_h = logging.FileHandler('app.log')
file_h.setLevel(logging.DEBUG)
file_h.setFormatter(logging.Formatter(
    '%(asctime)s %(name)s %(levelname)s: %(message)s'
))

logger.addHandler(console)
logger.addHandler(file_h)

logger.debug('This goes to file only')
logger.info('This goes to both')
logger.error('Error in both')
Output
INFO: This goes to both
ERROR: Error in both
(file gets all three with timestamps)

Interview Mastery

Q:

Why use logging over print statements?

A:

Logging provides: severity levels (filter noise), timestamps, source file/line info, multiple outputs (console + file), ability to disable all logs in production with one line, structured formatting, and is thread-safe. print() gives none of these.
