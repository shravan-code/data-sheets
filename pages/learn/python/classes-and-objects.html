Classes & Objects

Python is an object-oriented programming language. Classes are blueprints for creating objects (instances). They encapsulate data (attributes) and behavior (methods) into a single logical unit.

The Class Blueprint

A class is defined using the `class` keyword. It acts as a template. When you call the class, you create an 'instance' of that class.

class User:
    pass

# Creating instances
user1 = User()
user2 = User()

print(type(user1))  # <class '__main__.User'>
print(user1 == user2) # False (different objects in memory)
Output
<class '__main__.User'>
False

The __init__ Method & self

`__init__` is the constructor method called automatically when an object is created. `self` refers to the specific instance being created, allowing you to attach data to it.

class User:
    def __init__(self, username, email):
        self.username = username  # Instance attribute
        self.email = email        # Instance attribute
        self.is_active = True

    def greet(self):
        return f'Hello, I am {self.username}'

user = User('johndoe', 'john@example.com')
print(user.username)
print(user.greet())
Output
johndoe
Hello, I am johndoe

Attributes: Class vs Instance

Instance attributes are unique to each object. Class attributes are shared across all instances of the class.

class Employee:
    company = 'Data Cake'  # Class Attribute

    def __init__(self, name):
        self.name = name   # Instance Attribute

emp1 = Employee('Alice')
emp2 = Employee('Bob')

print(emp1.company)  # Data Cake
print(emp2.company)  # Data Cake

# Changing class attribute affects all
Employee.company = 'Cake Tech'
print(emp1.company)  # Cake Tech

Methods: Instance, Class, and Static

1. Instance Methods: Take `self`. Access instance data. 2. Class Methods: Take `cls`. Access class-level data. 3. Static Methods: Take nothing. Pure utility functions.

class Calculator:
    brand = 'Casio'

    def __init__(self, version):
        self.version = version

    def get_version(self):
        return f'Version: {self.version}'

    @classmethod
    def get_brand(cls):
        return f'Brand: {cls.brand}'

    @staticmethod
    def add(a, b):
        return a + b

calc = Calculator('v2.0')
print(calc.get_version()) # Instance access
print(Calculator.get_brand()) # Class access
print(Calculator.add(5, 10))  # Static access
Output
Version: v2.0
Brand: Casio
15

Interview Mastery

Q:

What is the difference between @classmethod, @staticmethod, and instance method?

A:

Instance method receives `self` (instance). @classmethod receives `cls` (class) — used for factory methods, alternative constructors. @staticmethod receives nothing special — it's a regular function in the class namespace, used for utility functions logically related to the class.

Q:

What is the purpose of 'self' in Python?

A:

'self' represents the instance of the class. By using 'self', we can access the attributes and methods of the class in python. It binds the attributes with the given arguments. Python passes it automatically as the first argument when calling an instance method.
