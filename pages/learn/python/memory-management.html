Memory Management

Reference Counting and Garbage Collection

CPython uses reference counting as primary memory management, with a cyclic garbage collector for cycles.

import sys
import gc

# Reference counting
a = [1, 2, 3]
print(sys.getrefcount(a))  # 2 (a + getrefcount arg)
b = a
print(sys.getrefcount(a))  # 3
del b
print(sys.getrefcount(a))  # 2

# Object size
print(sys.getsizeof([]))          # 56
print(sys.getsizeof([1,2,3]))     # 88
print(sys.getsizeof('hello'))     # 54
print(sys.getsizeof({}))          # 64

# Garbage collector (handles cycles)
class Node:
    def __init__(self):
        self.ref = None

a = Node()
b = Node()
a.ref = b  # a -> b
b.ref = a  # b -> a (cycle!)

del a, b
print(gc.collect())   # collects cyclic garbage

# GC settings
print(gc.get_threshold())  # (700, 10, 10) - generation thresholds
print(gc.get_count())      # objects in each generation

__slots__ — Memory Optimization

__slots__ replaces per-instance __dict__ with fixed-size array, reducing memory significantly.

import sys

# Normal class - uses __dict__
class PointDict:
    def __init__(self, x, y):
        self.x = x
        self.y = y

# Slots class - no __dict__
class PointSlots:
    __slots__ = ('x', 'y')
    def __init__(self, x, y):
        self.x = x
        self.y = y

d = PointDict(1, 2)
s = PointSlots(1, 2)

print(sys.getsizeof(d))          # ~48
print(sys.getsizeof(s))          # ~56 (smaller for many instances)
print(hasattr(d, '__dict__'))    # True
print(hasattr(s, '__dict__'))    # False

# Memory for 1M objects
import tracemalloc
tracemalloc.start()

points_dict  = [PointDict(i, i) for i in range(100000)]
current, peak = tracemalloc.get_traced_memory()
print(f'Dict: {peak/1024/1024:.1f} MB')
tracemalloc.reset_peak()

points_slots = [PointSlots(i, i) for i in range(100000)]
current, peak = tracemalloc.get_traced_memory()
print(f'Slots: {peak/1024/1024:.1f} MB')

Object Interning

Python caches small integers (-5 to 256) and interned strings for memory efficiency.

# Integer interning (-5 to 256)
a = 256
b = 256
print(a is b)   # True (same object)

c = 257
d = 257
print(c is d)   # False (different objects)

# String interning
s1 = 'hello'
s2 = 'hello'
print(s1 is s2)  # True (interned automatically)

s3 = 'hello world'  # may or may not be interned
s4 = 'hello world'
print(s3 is s4)     # implementation-dependent

# Force intern
import sys
s5 = sys.intern('force interned string')
s6 = sys.intern('force interned string')
print(s5 is s6)  # True
Output
True
False
True
False
True

Interview Mastery

Q:

How does Python manage memory?

A:

CPython uses reference counting: each object tracks how many references point to it. When count hits 0, memory is freed immediately. For cyclic references (A->B->A), a generational garbage collector runs periodically using gc module. Python also has a private heap and memory pools (pymalloc) for small objects.

Q:

When would you use __slots__?

A:

Use __slots__ when creating many instances of a class (millions) to reduce memory. It prevents dynamic attribute creation (no __dict__), reduces per-instance memory by ~30-50%, and slightly speeds up attribute access. Downside: cannot add arbitrary attributes, complicates multiple inheritance.
