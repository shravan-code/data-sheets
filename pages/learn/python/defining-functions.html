Section 4: Programming

Defining Functions

Arguments: Positional & Keyword

Standard arguments where order matters (positional) or names are specified (keyword). Keyword arguments improve readability.

def greet(name, greeting='Hello'):
    return f'{greeting}, {name}!'

print(greet('Alice'))              # positional
print(greet(name='Bob', greeting='Hi')) # keyword
Output
Hello, Alice!
Hi, Bob!

Variable Arguments: *args

Accept any number of positional arguments as a tuple. Perfect for functions like sum() or join().

def sum_all(*args):
    return sum(args)

print(sum_all(1, 2, 3, 4, 5))
Output
15

Variable Keyword Arguments: **kwargs

Accept any number of named arguments as a dictionary. Useful for handling optional configurations or passing settings.

def print_config(**kwargs):
    for key, value in kwargs.items():
        print(f'{key}: {value}')

print_config(host='localhost', port=8080)
Output
host: localhost
port: 8080

Keyword-only and Positional-only Parameters

* forces keyword-only; / forces positional-only (Python 3.8+).

# Keyword-only: params after *
def config(host, port, *, debug=False, timeout=30):
    print(host, port, debug, timeout)

config('localhost', 8080, debug=True)
# config('localhost', 8080, True)  # TypeError!

# Positional-only: params before /
def add(a, b, /):
    return a + b
add(1, 2)       # OK
# add(a=1, b=2)  # TypeError!

# Both
def full(pos_only, /, normal, *, kw_only):
    print(pos_only, normal, kw_only)
full(1, 2, kw_only=3)
full(1, normal=2, kw_only=3)
Output
localhost 8080 True 30
1 2 3
1 2 3

Returning Multiple Values

Python returns a tuple when using multiple return values.

def min_max(lst):
    return min(lst), max(lst)  # returns tuple

result = min_max([3, 1, 4, 1, 5, 9])
print(result)           # (1, 9)
print(type(result))     # tuple

# Unpack directly
low, high = min_max([3, 1, 4, 1, 5, 9])
print(low, high)
Output
(1, 9)
<class 'tuple'>
1 9

Interview Mastery

Q:

What is the mutable default argument trap?

A:

Default arguments are evaluated ONCE at function definition, not each call. `def f(lst=[]):` shares the same list across calls. Fix: `def f(lst=None): if lst is None: lst = []`.
