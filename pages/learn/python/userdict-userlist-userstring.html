UserDict, UserList, UserString

Custom Container Base Classes

Use UserDict/UserList/UserString when subclassing built-in containers to avoid C-level gotchas.

from collections import UserDict, UserList, UserString

# Custom dict that normalizes keys to lowercase
class CaseInsensitiveDict(UserDict):
    def __setitem__(self, key, value):
        super().__setitem__(key.lower(), value)
    
    def __getitem__(self, key):
        return super().__getitem__(key.lower())
    
    def __contains__(self, key):
        return super().__contains__(key.lower())

d = CaseInsensitiveDict()
d['Name'] = 'Alice'
d['AGE'] = 30
print(d['name'])   # Alice
print(d['age'])    # 30
print('NAME' in d)  # True

# Custom list with type enforcement
class TypedList(UserList):
    def __init__(self, typ, data=None):
        self.typ = typ
        super().__init__(data or [])
    
    def append(self, item):
        if not isinstance(item, self.typ):
            raise TypeError(f'Expected {self.typ.__name__}')
        super().append(item)

ints = TypedList(int)
ints.append(1)
ints.append(2)
print(ints.data)
try:
    ints.append('three')  # TypeError
except TypeError as e:
    print(e)
Output
Alice
30
True
[1, 2]
Expected int

Interview Mastery

Q:

Why use UserDict instead of subclassing dict directly?

A:

When subclassing dict directly, some built-in methods (like update, __init__) don't call your overridden __setitem__, causing inconsistent behavior. UserDict stores data in self.data (a plain dict) and routes all operations through Python methods, so overrides work correctly every time.
