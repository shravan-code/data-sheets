High-Level Language

Python is a high-level language, meaning it abstracts away complex details like memory management and CPU instructions. It uses English-like syntax, making it more readable for humans and allowing developers to focus on 'what' to do rather than 'how' the hardware should do it.

Interpreted vs Compiled

In Compiled languages (C, C++), code is converted to machine code before running. In Interpreted languages (PHP, Ruby), code is executed line-by-line. Python is a hybrid: Source code (.py) is first compiled into intermediate Bytecode (.pyc), which is then interpreted by the Python Virtual Machine (PVM).

Global Interpreter Lock (GIL)

The GIL is a mutex that protects access to Python objects, preventing multiple threads from executing Python bytecodes at once. While it makes CPython's memory management simpler, it means that even on multi-core systems, only one thread runs at a time for CPU-bound tasks.

Dynamic Typing

Python is dynamically typed, meaning variable types are determined at runtime. You don't need to declare whether a variable is an int, str, or list; the interpreter figures it out based on the value assigned. This allows for rapid development but requires discipline to avoid type errors.

Execution Flow: Source to Machine

Python's execution flow involves a compilation step to bytecode followed by interpretation by the PVM. This hybrid approach provides portability and ease of development.

Bytecode vs Machine Code

Bytecode is a low-level, platform-independent set of instructions for the PVM. Machine Code is the binary code (0s and 1s) specific to your computer's hardware (e.g., Intel vs. ARM). You can view Python's bytecode using the 'dis' module.

import dis

def add(a, b):
    return a + b

# Display the bytecode instructions
dis.dis(add)
Output
  2           0 LOAD_FAST                0 (a)
              2 LOAD_FAST                1 (b)
              4 BINARY_ADD
              6 RETURN_VALUE

Interview Mastery

Q:

What is the GIL in Python?

A:

The Global Interpreter Lock (GIL) is a mutex in CPython that allows only one thread to execute Python bytecode at a time. It simplifies memory management but limits multi-threaded CPU-bound performance. Use multiprocessing or async for parallelism.

Q:

Is Python interpreted or compiled?

A:

Python is both. Source code is compiled to bytecode (.pyc files) which is then interpreted by the CPython virtual machine. This is why Python is often called 'interpreted' but technically it has a compilation step.
