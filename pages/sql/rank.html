Section 12: Window Functions

RANK

Base Table Context: sales

rep region month amount
Alice East Jan 5000
Alice East Feb 7000
Bob West Jan 3000
Bob West Feb 6000
Carol East Jan 4000
Carol East Feb 8000

Window functions perform calculations across a set of table rows that are somehow related to the current row. Unlike aggregate functions, they do not group rows into a single output row; they retain the individual row identity while providing context from the surrounding 'window'.

SELECT rep, amount,
  RANK() OVER (ORDER BY amount DESC) AS rank
FROM sales;
Output Result
rep amount rank
Carol 8000 1
Alice 7000 2
Bob 6000 3
Alice 5000 4
Carol 4000 5
Bob 3000 6

Using RANK for analysis

SELECT 
  employee_name, 
  department, 
  salary, 
  RANK() OVER(PARTITION BY department ORDER BY salary DESC) as result 
FROM employees;
Output
Calculates the window value for each row within its partition.
Pros / Advantages
  • Complex analytical queries without self-joins
  • Retains row-level detail while calculating aggregates
Cons / Limitations
  • High memory/CPU usage on very large sets
  • Not supported in very old database versions
Pro Tips & Tricks
  • Use PARTITION BY to scope your calculations
  • Leverage LEAD/LAG for time-series analysis

Troubleshooting & Common Issues

Issue: Performance Lag
Solution: Ensure the ORDER BY column inside OVER() is indexed.
