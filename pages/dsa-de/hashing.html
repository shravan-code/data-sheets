4. Hashing

HashMap, HashSet, Frequency Count, Two-Sum Pattern.

HashMap & HashSet

A HashMap stores Key-Value pairs. It uses a mathematical hashing function to turn a key into an index, allowing for instant O(1) lookups. A HashSet is like a HashMap but only stores unique keys.

flowchart LR Key["Key: 'apple'"] --> HashFunction(Hash Function) HashFunction --> Index[Index: 3] Index --> Memory[Bucket 3: Value=50]

Core Concepts

  • Hashing: The process of converting a key into a fixed-size integer.
  • Collision: When two different keys result in the same hash. Modern dictionaries handle this automatically.
  • O(1) Lookups: Finding data in a hash map doesn't depend on how many items are in it.

The Two-Sum Pattern

# Problem: Find indices of two numbers that add up to target
def two_sum(nums, target):
    seen = {} # Number: Index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []

print(two_sum([2, 7, 11, 15], 9)) # Output: [0, 1]

Frequency Count Pattern

What it solves: Count occurrences, validate conditions like “same characters”, or group items by value.

Time: O(N) to build the map, space: O(N) in worst case.

def frequency_count(nums):
    freq = {}
    for x in nums:
        freq[x] = freq.get(x, 0) + 1
    return freq

# Input
nums = [1, 1, 2, 3, 2, 1]

# Output
print(frequency_count(nums))  # {1: 3, 2: 2, 3: 1}

HashSet Pattern (unique / presence checks)

Use HashSet when: you only care whether an element exists, or you want to remove duplicates quickly.

Time: O(N) average, with O(N) extra space.

def contains_duplicate(nums):
    seen = set()
    for x in nums:
        if x in seen:
            return True
        seen.add(x)
    return False

# Input
nums = [1, 2, 3, 1]

# Output
print(contains_duplicate(nums))  # True
