3. Variables & Data Types

How to declare, type, and use variables. Understanding PowerShell's rich type system.

Variable Declaration

Variables in PowerShell start with $. They are dynamically typed but can be explicitly typed. Special variables like $null, $true, $false are built-in.

Declaring Variables

$name = 'Alice'
$age = 30
$isAdmin = $true
$nothing = $null

# Strongly typed variables
[int]$count = 10
[string]$message = 'Hello'
[bool]$flag = $false
[double]$pi = 3.14159

# Check variable type
$name.GetType()
$age.GetType().FullName

Special Variables

$_          # Current pipeline object
$$          # Last token in previous command
$?          # Status of last operation (True/False)
$LASTEXITCODE  # Exit code of last native command
$PSScriptRoot  # Directory of current script
$PSCommandPath # Full path of current script
$HOME       # User's home directory
$env:PATH   # Environment variable PATH
$args       # Arguments passed to a script
$input      # Pipeline input object
$MyInvocation  # Info about current command

Strings

PowerShell has two types of strings: single-quoted (literal) and double-quoted (interpolated). Here-strings support multi-line text.

String Types

$user = 'Bob'

# Single-quoted: literal, no interpolation
$s1 = 'Hello $user'     # Output: Hello $user

# Double-quoted: interpolation
$s2 = "Hello $user"     # Output: Hello Bob
$s3 = "2 + 2 = $(2+2)" # Subexpression: 2 + 2 = 4

# Here-strings (multi-line)
$text = @'
This is a
multi-line string
with $user not interpolated
'@

$html = @"
<html>
  <body>Hello, $user</body>
</html>
"@

String Methods

$str = 'Hello, PowerShell World!'

$str.ToUpper()             # HELLO, POWERSHELL WORLD!
$str.ToLower()             # hello, powershell world!
$str.Length                # 21
$str.Replace('World','PS') # Hello, PowerShell PS!
$str.Split(',')
$str.Trim()
$str.TrimStart()
$str.TrimEnd()
$str.StartsWith('Hello')   # True
$str.EndsWith('!')
$str.Contains('PowerShell')
$str.IndexOf('P')          # 7
$str.Substring(7, 10)      # PowerShell

# String formatting
'{0} is {1} years old' -f 'Alice', 30
[string]::Format('{0:N2}', 1234.5678)

Arrays

Arrays store multiple values. PowerShell arrays are fixed-size by default; use ArrayList or generic List for dynamic sizing.

Array Operations

# Create arrays
$arr = 1, 2, 3, 4, 5
$arr = @(1, 2, 3)
$empty = @()
$single = @(42)

# Access elements
$arr[0]        # First element: 1
$arr[-1]       # Last element: 5
$arr[1..3]     # Slice: 2, 3, 4

# Array properties
$arr.Length
$arr.Count

# Add to array (creates new array)
$arr += 6

# ArrayList (dynamic)
$list = [System.Collections.ArrayList]@()
$list.Add('apple')
$list.Add('banana')
$list.Remove('apple')
$list.Insert(0, 'cherry')

# Array methods
$arr -contains 3      # True
$arr.IndexOf(3)
[array]::Sort($arr)
[array]::Reverse($arr)

Multi-Dimensional Arrays

# Jagged array
$matrix = @(
    @(1, 2, 3),
    @(4, 5, 6),
    @(7, 8, 9)
)
$matrix[0][1]  # 2

# .NET 2D array
$grid = New-Object 'int[,]' 3, 3
$grid[0,0] = 1
$grid[1,1] = 5
$grid[2,2] = 9

Hash Tables

Hash tables (dictionaries) store key-value pairs. They are widely used in PowerShell for structured data and splatting.

Hash Table Operations

# Create hashtable
$person = @{
    Name    = 'Alice'
    Age     = 30
    City    = 'New York'
}

# Access values
$person['Name']
$person.Name
$person.Age

# Add/Update/Remove
$person['Email'] = 'alice@example.com'
$person.Phone = '555-1234'
$person.Remove('Phone')

# Check existence
$person.ContainsKey('Name')
$person.ContainsValue('Alice')

# Iterate
foreach ($key in $person.Keys) {
    Write-Host "$key = $($person[$key])"
}

# Ordered hashtable
$ordered = [ordered]@{
    First  = 1
    Second = 2
    Third  = 3
}

# Splatting parameters
$params = @{
    Path        = 'C:\Logs'
    Recurse     = $true
    Filter      = '*.log'
}
Get-ChildItem @params
