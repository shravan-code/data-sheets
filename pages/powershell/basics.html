2. Basic Syntax & Fundamentals

Core syntax rules, cmdlet structure, and fundamental concepts every PowerShell scripter must know.

Cmdlet Structure

PowerShell commands (cmdlets) follow a Verb-Noun naming convention. They accept parameters and return .NET objects.

Basic Cmdlet Usage

# Verb-Noun structure
Get-Process
Stop-Process -Name notepad
Start-Service -Name wuauserv

# Common Verbs: Get, Set, New, Remove, Start, Stop, Invoke, Test, Add

Aliases

# Many cmdlets have aliases
ls          # Alias for Get-ChildItem
dir         # Alias for Get-ChildItem
cd          # Alias for Set-Location
cat         # Alias for Get-Content
ps          # Alias for Get-Process

# List all aliases
Get-Alias
Get-Alias ls

Comments

PowerShell supports single-line and multi-line (block) comments.

Comment Syntax

# This is a single-line comment

<#
  This is a
  multi-line comment block
#>

# Inline comment
$x = 5  # This sets x to 5

Semicolons & Line Continuation

Multiple statements can be written on one line using semicolons. Long lines can be split using backtick.

Multiple Statements & Line Continuation

$a = 1; $b = 2; $c = $a + $b

# Line continuation with backtick
Get-Process |
    Where-Object { $_.CPU -gt 100 } |
    Sort-Object CPU -Descending

# Backtick continuation
$result = Get-Process `
    -Name 'explorer' `
    -ErrorAction SilentlyContinue
