Modern Data Stack Architecture

The Modern Data Stack (MDS) is a cloud-native, modular data architecture using best-of-breed SaaS tools for each layer.

🏗

Layers

Ingestion["Fivetran","Airbyte","Stitch"]…
Storage["Snowflake","BigQuery","Redshift"]…
Transformation["dbt Core / Cloud"]…
Orchestration["Airflow","Prefect","Dagster"]…
BI["Looker","Tableau","Mode","Metabase"]…
Reverse Etl["Census","Hightouch"]…
Observability["Monte Carlo","Soda"]…
Catalog["DataHub","Alation"]…
📌

Example Stack

CompanyB2B SaaS startup
IngestionFivetran syncs Salesforce, Stripe, PostgreSQL → Snowflake
Transformationdbt models: staging → intermediate → marts
BiLooker dashboards for product and finance teams
Reverse EtlCensus syncs churn scores from Snowflake back to Salesforce
⚠️

Drawbacks

  • •Vendor dependency and SaaS costs add up quickly
  • •Data always flows through third-party tools — security concern
  • •Less control over performance tuning vs self-managed stack

Lakehouse Architecture on Databricks

🏗

Layers

IngestionAuto Loader (stream from S3/ADLS) / Kafka connector
BronzeDelta Lake raw tables — append-only, partitioned by date
SilverDelta Live Tables (DLT) pipelines — cleanse, deduplicate, merge
Golddbt models on Databricks SQL or DLT expectations
MlMLflow tracking + Feature Store on Gold Delta tables
ServingDatabricks SQL Warehouse (for BI) + Model Serving (for ML)
OrchestrationDatabricks Workflows (Jobs) or MWAA
📌
