Role-Based Access Control (RBAC)

Access to data is controlled by roles. Users are assigned roles. Roles have permissions. Easier to manage than per-user permissions.

💡

Example

Roles[{"role":"data_analyst","permissions":["SELECT on gold.*"]},{"role":"data_engine…
Snowflake SqlCREATE ROLE data_analyst; GRANT USAGE ON DATABASE analytics TO ROLE data_analyst; GRANT USAGE ON SCHEMA analytics.gold TO ROLE data_analyst; GRANT SELECT ON ALL TABLES IN SCHEMA analytics.gold TO ROLE data_analyst; GRANT ROLE data_analyst TO USER alice;

Column-Level & Row-Level Security

Fine-grained access control at the column or row level within a table.

📌

Column Level

DescriptionCertain columns (PII, salary) are masked or hidden for specific roles
Snowflake ExampleCREATE MASKING POLICY mask_email AS (val STRING) RETURNS STRING -> CASE WHEN CURRENT_ROLE() IN ('data_admin') THEN val ELSE REGEXP_REPLACE(val, '.+@', '****@') END; ALTER TABLE silver.customers MODIFY COLUMN email SET MASKING POLICY mask_email;
📌

Row Level

DescriptionUsers only see rows relevant to their region, department, or tenant
Snowflake ExampleCREATE ROW ACCESS POLICY region_policy AS (region_col STRING) RETURNS BOOLEAN -> CASE WHEN CURRENT_ROLE() = 'us_analyst' THEN region_col = 'US' WHEN CURRENT_ROLE() = 'eu_analyst' THEN region_col = 'EU' ELSE TRUE END; ALTER TABLE gold.sales ADD ROW ACCESS POLICY region_policy ON (region);

Network Security (VPC, Private Endpoints)

Isolate data infrastructure in private networks. Prevent public internet exposure.

📌

Vpc Architecture

Public SubnetLoad balancers, NAT gateways — exposed to internet
Private SubnetRDS, Kafka, Spark clusters — no direct internet access
Data SubnetS3, data lakes — accessed via VPC endpoints only
📌

Vpc Endpoints

DescriptionS3 Gateway Endpoint: Spark clusters in VPC connect to S3 without internet. No egress cost. Traffic stays on AWS network.
ExampleWithout VPC endpoint: Spark → Internet → S3 (egress cost + latency) With VPC endpoint: Spark → AWS internal network → S3 (no cost + lower latency)
📌

Security Groups

Inbound: only allow Spark cluster IPs to connect to Kafka (port 9092). Block all other IPs.
