Section 18: Foundations

Interview Master: Top 50 Python Interview Q&A

1 What is the difference between list, tuple, set, and dict?

list: ordered, mutable, allows duplicates, O(n) search. tuple: ordered, immutable, hashable, slightly faster. set: unordered, mutable, unique elements, O(1) search/add/remove. dict: key-value pairs, ordered (3.7+), O(1) key access. Use: list for sequences, tuple for fixed records, set for unique membership, dict for lookups.

2 Explain Python's memory management.

CPython uses reference counting: each object has a refcount. When it hits 0, memory is freed. A cyclic garbage collector (gc module) handles reference cycles. Python has a private heap managed by pymalloc for small objects. __slots__ reduces per-instance memory. del decrements refcount; sys.getrefcount() inspects it.

3 What is a decorator? Write one from scratch.

A decorator is a callable that takes a function and returns a modified function. Example: `def log(func): @wraps(func) def wrapper(*a,**kw): print(f'Calling {func.__name__}'); return func(*a,**kw); return wrapper`. Always use @functools.wraps to preserve __name__, __doc__.

4 What is the difference between shallow and deep copy?

Shallow copy (copy.copy / .copy() / [:]) copies the container but nested objects are still shared references. Deep copy (copy.deepcopy) recursively copies all nested objects. Example: list of lists — modifying a nested list in a shallow copy affects the original.

5 Explain generators and when to use them.

Generators are functions that yield values lazily. They use O(1) memory regardless of sequence size. Use when: processing large/infinite sequences, building data pipelines, values only needed once. Created with yield keyword or generator expressions (x for x in iterable).

6 What is the difference between @staticmethod and @classmethod?

@staticmethod receives no implicit first argument — it's a regular function in the class namespace. @classmethod receives cls (the class itself) as first argument. Use @classmethod for factory methods (alternate constructors), access to class state. Use @staticmethod for utility functions logically grouped with the class.

7 How does Python resolve multiple inheritance (MRO)?

Python uses C3 linearization (MRO). For class C(A, B): Python builds a list that preserves left-to-right order and ensures a class appears before its parents. Check with Class.__mro__. super() follows MRO, not just the direct parent.

8 What are *args and **kwargs?

*args captures extra positional args as a tuple. **kwargs captures extra keyword args as a dict. In a function signature, order must be: positional, *args, keyword-only, **kwargs. They can also unpack when calling: func(*list, **dict).

9 Explain the difference between is and ==.

== checks value equality (__eq__). is checks identity (same object in memory, same id()). Small integers (-5 to 256) and interned strings may share identity. Never use 'is' to compare values — always use == for equality, is only for None checks (x is None).

10 What is a context manager and how do you create one?

A context manager wraps setup/teardown around a with block. Implement __enter__ (returns resource) and __exit__ (cleanup, suppress exception if returns True). Or use @contextlib.contextmanager with a generator (yield is the body of the with block).

11 What is the GIL and when does it matter?

Global Interpreter Lock — CPython mutex allowing only one thread to run Python bytecode at once. Released during I/O operations. Matters for CPU-bound multithreaded code — threads don't parallelize. Solutions: multiprocessing (separate processes), asyncio (async I/O), or use C extensions that release the GIL (e.g. NumPy).

12 How do you handle exceptions properly?

Catch specific exceptions (not bare except). Use try/except/else/finally correctly. Re-raise with `raise` or chain with `raise X from Y`. Create custom exceptions for domain errors. Never swallow exceptions silently. Log with logger.exception() to preserve traceback.

13 What is the difference between append() and extend()?

append(x) adds x as a single element (even if x is a list — you get a nested list). extend(iterable) adds each element of the iterable individually. [1,2].append([3,4]) → [1,2,[3,4]]. [1,2].extend([3,4]) → [1,2,3,4].

14 Explain list comprehension vs generator expression vs map().

List comp: creates full list in memory, use when you need to reuse or index the result. Generator: lazy, one item at a time, O(1) memory, use for single-pass iteration. map(): applies function lazily, like a generator. For readability, list/gen comprehensions are preferred. map() is fast for simple built-in functions.

15 What is functools.lru_cache? When would you use it?

lru_cache memoizes a function's results based on arguments. Subsequent calls with same args return cached result. Use for: pure functions called repeatedly with same inputs, expensive computations (Fibonacci, DB lookups in read-only scenarios). Set maxsize=None for unlimited cache. Use cache() as shorthand in Python 3.9+.

16 How would you find duplicates in a list?

Using Counter: `[x for x, c in Counter(lst).items() if c > 1]`. Using sets: `seen = set(); dups = set(x for x in lst if x in seen or seen.add(x))`. For just checking existence: `len(lst) != len(set(lst))`.

17 What is the difference between module and package?

A module is a single .py file. A package is a directory containing an __init__.py (makes it a package) and possibly submodules/subpackages. Import with `import module` or `from package.module import name`.

18 Explain Python's LEGB scope rule.

Python resolves names in this order: Local (inside current function), Enclosing (outer function scopes), Global (module level), Built-in (Python builtins). Use `global` to assign to global scope, `nonlocal` to assign to enclosing function scope.

19 What is a metaclass?

A metaclass is the class of a class — it controls how classes are created. type is the default metaclass. Custom metaclasses inherit from type and override __new__ or __init__. Used for: ORM models, singleton enforcement, automatic method registration. Prefer __init_subclass__ or class decorators for simpler cases.

20 How do you make a class iterable?

Implement __iter__ (returns iterator) and __next__ (returns next value, raises StopIteration when done). Or implement __iter__ as a generator function (uses yield). Or __getitem__ with integer indices (legacy protocol).

21 What is the difference between sorted() and list.sort()?

sorted() works on any iterable, returns a new list, original unchanged. list.sort() modifies in-place, returns None, only works on lists. Both accept key= and reverse= parameters. list.sort() is slightly faster (no copy).

22 How do you reverse a list? What are the different approaches?

1) lst[::-1] — creates new reversed list. 2) list(reversed(lst)) — lazy reversed() iterator. 3) lst.reverse() — in-place, returns None. For strings: s[::-1]. For generators: can't reverse directly, convert to list first.

23 Explain Python's data model (dunder methods).

Python's data model lets custom classes integrate with language features via __dunder__ methods. __len__ enables len(), __iter__ enables for loops, __add__ enables +, __enter__/__exit__ enables with. This is why Python's built-in functions and operators work on custom objects — they call these protocols.

24 What is the walrus operator (:=)?

The walrus operator (Python 3.8+) assigns and returns a value in a single expression. Useful in while loops: `while chunk := f.read(8192): process(chunk)` or comprehensions: `[y for x in data if (y := f(x)) > 0]`. Avoids calling the same function twice.

25 How do you handle circular imports in Python?

Options: 1) Restructure code to remove cycle. 2) Move import inside the function that needs it (lazy import). 3) Import the module, not the name: `import module; module.name`. 4) Use TYPE_CHECKING guard for type-hint-only imports. Root cause is usually poor module structure.

26 What is the GIL in Python?

The Global Interpreter Lock (GIL) is a mutex in CPython that allows only one thread to execute Python bytecode at a time. It simplifies memory management but limits multi-threaded CPU-bound performance. Use multiprocessing or async for parallelism.

27 Is Python interpreted or compiled?

Python is both. Source code is compiled to bytecode (.pyc files) which is then interpreted by the CPython virtual machine. This is why Python is often called 'interpreted' but technically it has a compilation step.

28 What is the difference between mutable and immutable types?

Mutable objects (list, dict, set) can be changed after creation. Immutable objects (int, str, tuple, frozenset) cannot. Immutability makes objects hashable (usable as dict keys). When you 'modify' an immutable, Python creates a new object.

29 What does `is` vs `==` check?

`==` checks value equality. `is` checks identity (same object in memory via id()). Example: `a = [1,2]; b = [1,2]; a == b` is True but `a is b` is False. Small integers (-5 to 256) and interned strings may share identity due to caching.

30 What is operator precedence in Python?

From highest to lowest: ** (exponent), ~+- (unary), */%// , +-, <<>>, &, ^, |, comparisons, not, and, or. Use parentheses to make intent explicit.

31 How does Python's `or` operator work for default values?

`x = value or default` returns `value` if truthy, else `default`. But beware: if value is 0, '', [], etc. (valid but falsy), it returns default unexpectedly. Prefer `x = value if value is not None else default` for precision.

32 Why is string concatenation in a loop bad? What's the alternative?

Strings are immutable, so each `s += x` creates a new string object — O(n²) total. Use `''.join(list_of_strings)` which is O(n). Example: `result = ''.join(str(i) for i in range(1000))`.

33 What's the difference between find() and index()?

`find()` returns -1 if substring not found. `index()` raises a `ValueError`. Use `find()` when absence is a valid case; use `index()` when absence means a bug.

34 How do f-strings differ from .format()?

f-strings (3.6+) are evaluated at runtime and generally faster. They support expressions inline: `f'{2+2}'`. `.format()` is more flexible for templates stored as strings since f-strings must be defined inline. f-strings also support `=` for debugging: `f'{var=}'`.

35 What is the difference between `if x:` and `if x is not None:`?

`if x:` is False for None, 0, '', [], {} etc. `if x is not None:` is only False for None. Use the latter when 0 or empty string is a valid value you want to keep.

36 What is the for-else construct in Python?

The `else` block of a `for` or `while` loop runs only if the loop completes without hitting a `break`. Useful for search algorithms: loop to find item, break if found, else block handles 'not found' case.

37 What's the difference between range() and list(range())?

`range()` returns a lazy range object (O(1) memory regardless of size). `list(range(n))` materializes it into a list (O(n) memory). Always prefer `range()` in for loops.

38 When should you NOT use list comprehensions?

Avoid when: (1) logic is complex — use a regular loop for readability, (2) you need side effects like printing (use a loop), (3) you only need to iterate once — use a generator expression instead to save memory.

39 What is the difference between a generator expression and a list comprehension?

List comprehension creates the full list in memory immediately. Generator expression is lazy — yields one item at a time. Use generators for large datasets or infinite sequences. Generators are also faster when you don't need all items (e.g., using `any()` or `next()`).

40 What is the time complexity of list operations?

append: O(1) amortized, pop from end: O(1), pop from index: O(n), insert at index: O(n), index lookup by index: O(1), search by value: O(n), sort: O(n log n).

41 How is list.sort() different from sorted()?

list.sort() modifies the list in-place and returns None. sorted() returns a new list and works on any iterable. Both accept key and reverse parameters. list.sort() is slightly faster since it doesn't create a new list.

42 Why use tuples instead of lists?

Tuples: (1) are immutable so hashable — usable as dict keys or set elements, (2) slightly faster than lists due to fixed size, (3) signal intent that data shouldn't change (e.g., coordinates, RGB values), (4) can be used for multiple return values.

43 What is the time complexity of dict operations?

Average O(1) for get, set, delete, and `in` operator. Worst case O(n) due to hash collisions (rare). Iteration is O(n).

44 How would you count word frequency in a sentence?

Use Counter: `from collections import Counter; freq = Counter(sentence.split())`. Or manually: `freq = {}; freq[word] = freq.get(word, 0) + 1`.

45 How would you remove duplicates from a list while preserving order?

Python 3.7+: `list(dict.fromkeys(lst))` — dicts preserve insertion order. Or: `seen = set(); result = [x for x in lst if not (x in seen or seen.add(x))]`. Simple `list(set(lst))` removes duplicates but loses order.

46 What is the mutable default argument trap?

Default arguments are evaluated ONCE at function definition, not each call. `def f(lst=[]):` shares the same list across calls. Fix: `def f(lst=None): if lst is None: lst = []`.

47 Explain the closure variable capture gotcha in loops.

Functions in a loop capture the variable, not its value: `funcs = [lambda: i for i in range(3)]` — all return 2 (last value). Fix: use default arg: `lambda i=i: i` or `functools.partial`.

48 Why use @functools.wraps in decorators?

Without it, the wrapper function replaces the original, losing __name__, __doc__, __module__, etc. This breaks introspection, documentation tools, and debugging. @wraps copies these attributes from the wrapped function to the wrapper.

49 What is the difference between a decorator and a context manager?

Decorators modify or wrap a function's behavior permanently. Context managers manage resources for a block of code using with statement — they guarantee setup/teardown (like file closing). Both can be implemented similarly but serve different use cases.

50 What is the difference between yield and return?

return exits the function completely. yield pauses execution, returns a value, and resumes from the same point on the next call to next(). A function with yield is a generator — calling it returns a generator object, not the result.

51 How would you implement an infinite sequence in Python?

Use a generator: `def naturals(): n=0; while True: yield n; n+=1`. Generators are lazy so they don't compute all values. Combine with itertools.islice to take a finite portion.

52 What is the difference between @classmethod, @staticmethod, and instance method?

Instance method receives `self` (instance). @classmethod receives `cls` (class) — used for factory methods, alternative constructors. @staticmethod receives nothing special — it's a regular function in the class namespace, used for utility functions logically related to the class.

53 What is the purpose of 'self' in Python?

'self' represents the instance of the class. By using 'self', we can access the attributes and methods of the class in python. It binds the attributes with the given arguments. Python passes it automatically as the first argument when calling an instance method.

54 What is the MRO and how does Python resolve method calls in multiple inheritance?

MRO (Method Resolution Order) determines which class's method is called. Python uses the C3 linearization algorithm: left-to-right depth-first with no duplicates. Check with `ClassName.__mro__`. super() follows the MRO, not necessarily the direct parent.

55 What is the difference between __str__ and __repr__?

__str__ is for human-readable output (used by print/str()). __repr__ is for unambiguous developer representation — should ideally return code that recreates the object. If __str__ is not defined, Python falls back to __repr__. Always define __repr__ as minimum.

56 When does Python call __eq__ vs __hash__?

If you override __eq__, Python sets __hash__ to None (making the object unhashable) unless you also define __hash__. Objects that compare equal must have the same hash. The contract: `a == b` implies `hash(a) == hash(b)`.

57 Why use default_factory instead of a mutable default?

Mutable defaults (like `field: list = []`) are shared across all instances — a classic Python bug. `default_factory=list` calls `list()` for each new instance, giving each its own list. This mirrors the mutable default argument trap.

58 When to use dataclass vs NamedTuple vs plain class?

Use dataclass for mutable data with behavior, validation, and flexibility. Use NamedTuple for immutable records that need tuple-like unpacking. Use plain class for complex logic. Dataclasses are the modern default for simple data containers.

59 What is the difference between except Exception and except BaseException?

except Exception catches all normal exceptions. except BaseException also catches SystemExit, KeyboardInterrupt, GeneratorExit — which you usually DON'T want to catch. Always use except Exception unless you have a specific reason to catch system-level events.

60 When does the else block in try/except run?

The else block runs only if the try block completed without raising any exception. It's used to separate 'code that might fail' from 'code that should run on success', keeping the except block focused on actual error handling.

61 Why create custom exceptions?

Custom exceptions allow callers to catch specific error types from your library without catching unrelated built-in exceptions. They carry domain-specific context (like field name in ValidationError), and enable building clean exception hierarchies that mirror your application's domain.

62 Why use logging over print statements?

Logging provides: severity levels (filter noise), timestamps, source file/line info, multiple outputs (console + file), ability to disable all logs in production with one line, structured formatting, and is thread-safe. print() gives none of these.

63 What is the most common mistake with groupby?

Forgetting to sort the data first. groupby only groups CONSECUTIVE elements with the same key — if unsorted, you get multiple groups for the same key. Always: `sorted(data, key=keyfunc)` before `groupby(data, key=keyfunc)`.

64 How would you generate all subsets of a list?

Using combinations across all lengths: `from itertools import combinations; subsets = [c for r in range(len(lst)+1) for c in combinations(lst, r)]`. This gives all 2^n subsets.

65 When would you use functools.partial vs a lambda?

partial is better when: freezing named arguments for reuse, creating multiple variants of a function, when the result needs to be inspected (.func, .args, .keywords). Lambda is better for quick one-off transformations inline. partial also works correctly with pickle (lambdas don't).

66 What's the difference between lru_cache and cached_property?

lru_cache caches based on function arguments — same args returns cached result. cached_property caches per instance with no arguments — computed once on first access, stored in instance __dict__. Use cached_property for expensive computed attributes on objects.

67 What is the difference between Protocol and ABC?

ABC (Abstract Base Class) requires explicit inheritance: `class MyClass(MyABC)`. Protocol uses structural subtyping — if an object has the right methods/attributes, it satisfies the Protocol without inheritance. Protocol is the Pythonic way to formalize duck typing. ABCs are better for enforcing that subclasses implement required methods.

68 Does Python enforce type hints at runtime?

No. Type hints are purely for static analysis tools (mypy, pyright) and IDE support. Python ignores them at runtime unless you use a library like pydantic or beartype. You can access annotations via __annotations__ or get_type_hints().

69 When use deque over list?

Use deque when you need O(1) appendleft/popleft (list is O(n) for these). Typical use cases: queues (FIFO), sliding windows with maxlen, breadth-first search, implementing undo history. For random access by index, list is still faster.

70 Why use UserDict instead of subclassing dict directly?

When subclassing dict directly, some built-in methods (like update, __init__) don't call your overridden __setitem__, causing inconsistent behavior. UserDict stores data in self.data (a plain dict) and routes all operations through Python methods, so overrides work correctly every time.

71 Why use pathlib over os.path?

pathlib provides an OOP interface — paths are objects with methods, not strings. The / operator for joining is cleaner than os.path.join(). Methods like .read_text(), .write_text() simplify file I/O. Path objects work across OS. For new code, pathlib is always preferred.

72 What is the GIL and how does it affect threading?

The Global Interpreter Lock (GIL) allows only one thread to execute Python bytecode at a time in CPython. This means CPU-bound threads don't actually run in parallel. However, the GIL is released during I/O operations, so threading is effective for I/O-bound tasks. For CPU-bound parallelism, use multiprocessing.

73 What is a race condition and how do you prevent it?

A race condition occurs when multiple threads access and modify shared data concurrently, producing unpredictable results. Prevent with: Lock (mutex), RLock (reentrant lock), Semaphore (limit concurrent access), Queue (thread-safe communication), or using thread-local storage.

74 When use threading vs multiprocessing?

Threading: I/O-bound tasks (network requests, file operations, DB queries) where threads wait most of the time. Multiprocessing: CPU-bound tasks (number crunching, image processing, ML) where you need true parallelism. Multiprocessing has higher overhead (process creation, IPC) but bypasses the GIL.

75 What is the difference between asyncio, threading, and multiprocessing?

asyncio: single thread, cooperative multitasking via event loop — best for high-concurrency I/O (thousands of connections). Threading: multiple threads, preemptive — good for moderate I/O concurrency. Multiprocessing: multiple processes — only option for true CPU parallelism in CPython. asyncio has lowest overhead per concurrent unit.

76 What happens if you call a blocking function inside an async function?

It blocks the entire event loop — no other coroutines can run. Always use async alternatives (aiohttp instead of requests, aiofiles instead of open). For unavoidable blocking calls, use `await loop.run_in_executor(None, blocking_func)` to run in a thread pool.

77 How does Python manage memory?

CPython uses reference counting: each object tracks how many references point to it. When count hits 0, memory is freed immediately. For cyclic references (A->B->A), a generational garbage collector runs periodically using gc module. Python also has a private heap and memory pools (pymalloc) for small objects.

78 When would you use __slots__?

Use __slots__ when creating many instances of a class (millions) to reduce memory. It prevents dynamic attribute creation (no __dict__), reduces per-instance memory by ~30-50%, and slightly speeds up attribute access. Downside: cannot add arbitrary attributes, complicates multiple inheritance.

79 When would you use a metaclass?

Metaclasses are for 'class creation hooks': enforcing interface contracts, registering subclasses automatically (plugin systems), modifying class attributes at definition time, ORMs (like Django models). Most of the time, __init_subclass__, class decorators, or descriptors are simpler alternatives.

80 What is the difference between a data descriptor and non-data descriptor?

Data descriptor implements both __get__ and __set__ (or __delete__). Non-data descriptor implements only __get__. Data descriptors take priority over instance __dict__, so they always intercept access. Non-data descriptors are overridden by instance __dict__ entries. @property is a data descriptor; functions are non-data descriptors.

81 What should __exit__ return?

Return True to suppress exceptions raised in the with block. Return False (or None) to propagate them. exc_type, exc_val, exc_tb are None if no exception occurred. The finally clause in @contextmanager always runs, making cleanup guaranteed.

82 What is the difference between Mock and MagicMock?

Mock is a basic mock object. MagicMock is a subclass that pre-configures all magic methods (__len__, __iter__, __str__, __enter__, __exit__, etc.) making it work with len(), for loops, with statements, etc. Use MagicMock when the code under test calls magic methods on the mock.

83 What is the difference between pytest fixtures and setUp/tearDown?

pytest fixtures: composable (inject multiple), have scope (function/class/module/session), use yield for teardown, can be shared via conftest.py, and are more granular. setUp/tearDown run for every test in the class. pytest fixtures are more flexible and Pythonic.

84 What are the most impactful Python performance optimizations?

1) Use built-in functions (sum, map, filter) over loops — they're C-implemented. 2) Use sets for O(1) membership testing. 3) Use ''.join() not string concatenation. 4) Cache expensive results with lru_cache. 5) Use generators for large data. 6) Profile first — never guess. 7) For numeric heavy-lifting, use NumPy vectorization.

85 How do Python's first-class functions affect design patterns?

Many Gang of Four patterns become simpler in Python. Strategy pattern: pass a function instead of a strategy object. Command pattern: use callables. Observer: pass callbacks. Factory: use dict of classes. Python's first-class functions, closures, and decorators replace much boilerplate that OOP patterns require in Java/C++.

86 What is EAFP vs LBYL in Python?

EAFP (Easier to Ask Forgiveness than Permission): try/except — Pythonic. `try: x = d['key'] except KeyError: ...`. LBYL (Look Before You Leap): check before acting — `if 'key' in d: x = d['key']`. EAFP is generally preferred in Python since it avoids race conditions and is more readable for common cases.

87 What is SQL injection and how do you prevent it?

SQL injection is when user input is embedded directly in SQL strings, allowing attackers to modify queries. Prevent by ALWAYS using parameterized queries: `cursor.execute('SELECT * FROM users WHERE id = ?', (user_id,))` — NEVER f-strings or string concatenation for SQL.
