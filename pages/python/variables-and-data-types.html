Section 1: Foundations

Variables & Data Types

Python uses dynamic typing — variables don't have fixed types. Types are inferred at runtime and can change.

Basic Types: Numbers

Python handles integers (int), decimals (float), and complex numbers (complex) automatically. Perfect for scientific computing or financial calculations.

x = 42           # int
y = 3.14         # float
z = 2 + 3j       # complex
print(type(x), type(y), type(z))
Output
<class 'int'> <class 'float'> <class 'complex'>

Basic Types: Booleans & None

bool represents True/False; None represents the absence of a value (null). Critical for control flow and status tracking.

b = True         # bool
n = None         # NoneType
print(type(b), type(n))
Output
<class 'bool'> <class 'NoneType'>

Dynamic Typing

Variables can be reassigned to different types. Use type() and isinstance() to check types.

x = 10
print(type(x))   # int
x = 'hello'
print(type(x))   # str now

print(isinstance(42, int))       # True
print(isinstance(42, (int, float)))  # True - checks multiple
Output
<class 'int'>
<class 'str'>
True
True

Type Casting

Convert between types explicitly using constructor functions.

print(int('42'))        # str -> int
print(float('3.14'))    # str -> float
print(str(100))         # int -> str
print(bool(0))          # 0 -> False
print(bool(1))          # 1 -> True
print(bool(''))         # empty str -> False
print(bool('hello'))    # non-empty -> True
print(int(3.9))         # truncates, not rounds -> 3
print(list('abc'))      # str -> list of chars
Output
42
3.14
100
False
True
False
True
3
['a', 'b', 'c']

Mutable vs Immutable

Immutable types: int, float, str, tuple, frozenset, bytes. Mutable types: list, dict, set. Immutability affects assignment behavior and hashability.

# Immutable - reassignment creates new object
a = 'hello'
b = a
a = a + ' world'  # new string object
print(b)          # b unchanged

# Mutable - both point to same object
list1 = [1, 2, 3]
list2 = list1
list1.append(4)
print(list2)      # list2 also changed!
Output
hello
[1, 2, 3, 4]

Interview Mastery

Q: What is the difference between mutable and immutable types?

A: Mutable objects (list, dict, set) can be changed after creation. Immutable objects (int, str, tuple, frozenset) cannot. Immutability makes objects hashable (usable as dict keys). When you 'modify' an immutable, Python creates a new object.

Q: What does `is` vs `==` check?

A: `==` checks value equality. `is` checks identity (same object in memory via id()). Example: `a = [1,2]; b = [1,2]; a == b` is True but `a is b` is False. Small integers (-5 to 256) and interned strings may share identity due to caching.
