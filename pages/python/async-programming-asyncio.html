Section 12: Foundations

Async Programming (asyncio)

async/await Basics

Async programming uses a single thread with cooperative multitasking. Best for high-concurrency I/O.

import asyncio
import time

# Coroutine - defined with async def
async def fetch(url, delay):
    print(f'Fetching {url}')
    await asyncio.sleep(delay)  # non-blocking sleep
    return f'Response from {url}'

# Run coroutines sequentially
async def sequential():
    start = time.perf_counter()
    r1 = await fetch('site1.com', 1)
    r2 = await fetch('site2.com', 1)
    print(f'Sequential: {time.perf_counter()-start:.1f}s')
    return [r1, r2]

# Run concurrently with gather
async def concurrent():
    start = time.perf_counter()
    r1, r2, r3 = await asyncio.gather(
        fetch('site1.com', 1),
        fetch('site2.com', 1),
        fetch('site3.com', 1)
    )
    print(f'Concurrent: {time.perf_counter()-start:.1f}s')
    return [r1, r2, r3]

asyncio.run(sequential())
asyncio.run(concurrent())
Output
Fetching site1.com
Fetching site2.com
Sequential: 2.0s
Fetching site1.com
Fetching site2.com
Fetching site3.com
Concurrent: 1.0s

Tasks, gather, and asyncio patterns

Create tasks for concurrent execution, handle timeouts and cancellation.

import asyncio

async def work(name, delay):
    print(f'{name} started')
    await asyncio.sleep(delay)
    print(f'{name} done')
    return name

async def main():
    # create_task - schedule immediately
    t1 = asyncio.create_task(work('Task1', 2))
    t2 = asyncio.create_task(work('Task2', 1))
    
    # await both (order of completion may differ)
    results = await asyncio.gather(t1, t2)
    print(f'Results: {results}')
    
    # Timeout
    try:
        await asyncio.wait_for(work('Slow', 5), timeout=1.0)
    except asyncio.TimeoutError:
        print('Timed out!')
    
    # as_completed - handle as each finishes
    tasks = [work(f'Job{i}', 3-i) for i in range(3)]
    for coro in asyncio.as_completed(tasks):
        result = await coro
        print(f'Finished: {result}')

asyncio.run(main())

Timed out!

Job0 started

Finished: Job0

async for, async with, asyncio.Queue

Async versions of iterators and context managers.

import asyncio

# Async context manager
class AsyncDBConnection:
    async def __aenter__(self):
        print('Connecting to DB...')
        await asyncio.sleep(0.1)  # simulate connection
        return self
    
    async def __aexit__(self, *args):
        print('Closing DB connection')
    
    async def query(self, sql):
        await asyncio.sleep(0.1)
        return f'Results for: {sql}'

# Async iterator
class AsyncRange:
    def __init__(self, n):
        self.n = n
        self.i = 0
    
    def __aiter__(self):
        return self
    
    async def __anext__(self):
        if self.i >= self.n:
            raise StopAsyncIteration
        await asyncio.sleep(0)  # yield control
        val = self.i
        self.i += 1
        return val

# asyncio.Queue - producer/consumer
async def producer(queue):
    for i in range(5):
        await queue.put(i)
        await asyncio.sleep(0.1)
    await queue.put(None)  # sentinel

async def consumer(queue):
    while True:
        item = await queue.get()
        if item is None: break
        print(f'Consumed: {item}')

async def main():
    async with AsyncDBConnection() as db:
        result = await db.query('SELECT *')
        print(result)
    
    async for i in AsyncRange(3):
        print(f'Async iter: {i}')
    
    queue = asyncio.Queue()
    await asyncio.gather(producer(queue), consumer(queue))

asyncio.run(main())

Interview Mastery

Q: What is the difference between asyncio, threading, and multiprocessing?

A: asyncio: single thread, cooperative multitasking via event loop — best for high-concurrency I/O (thousands of connections). Threading: multiple threads, preemptive — good for moderate I/O concurrency. Multiprocessing: multiple processes — only option for true CPU parallelism in CPython. asyncio has lowest overhead per concurrent unit.

Q: What happens if you call a blocking function inside an async function?

A: It blocks the entire event loop — no other coroutines can run. Always use async alternatives (aiohttp instead of requests, aiofiles instead of open). For unavoidable blocking calls, use `await loop.run_in_executor(None, blocking_func)` to run in a thread pool.
